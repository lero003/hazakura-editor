# v1.12 OKF Starter Scaffold

Status: Active implementation (entry points landed; pin evolution is ongoing)
Scope: OKF v0.1 Draft 互換の明示的ひな形作成（starter + book-like）
Authority: High for v1.12 implementation
Last reviewed: 2026-07-15

## Decision

v1.11 は **読み取り中心**の OKF Draft Compatibility Preview としてローカル
TestFlight 候補まで完了した。v1.12 では、同じ Draft 契約の **出発点**として、
ユーザーが明示的に選んだ親フォルダ配下へ「よくある Markdown 群」を一度だけ
書き出す **OKF starter scaffold** を追加する。

これは:

- **A. OKF starter** — 最小の知識フォルダひな形
- **B. OKF 互換 book 雛形** — 章っぽい concept と目次リンク付きのよくある並び

までを対象とする。OKF 公式の book format、Hazakura Book Scope、章順管理、
whole-book プレビュー / EPUB export は **対象外**（v2 / e-book レーン）。

in-tree version は `1.12.0`。App Store / TestFlight 公開は別判断。

**Spec pin and upgrade process** (templates and integrity checks move together):
`docs/okf-spec-pin.md`. Upstream OKF の version / SPEC が変われば、ひな形資産と
点検モデルは **同じ pin 更新の一部として**差し替える前提である。

## Relation To v1.11

| | v1.11 | v1.12 |
|---|---|---|
| 操作 | 点検（read） | ひな形作成（write） |
| 契約 | 共通 pin（`docs/okf-spec-pin.md`） | 同じ pin の **互換出発点** |
| 正本 | ディスク上の既存 Markdown | 新規に置いた Markdown 群 |
| 自動 | なし | なし（明示操作のみ） |
| pin 更新時 | pure model / fixtures | scaffold assets / registry |

v1.11 の optional polish「静的な最小例」は、**見て終わり**ではなく **置ける**
形として本レーンに吸収する。点検 UI 自体の大規模改修はしない。

## User-visible Contract

1. ワークスペースが開いているときだけ有効。
2. ユーザーが明示的に次のいずれかを実行する:
   - Command Palette: **知識フォルダのひな形を作成（最小）** /
     **知識フォルダのひな形を作成（本っぽい章立て）**
   - フォルダ / ルートのコンテキストメニュー: 同趣旨の2項目
   - ワークスペースサイドバー **新規 (+)** メニュー: 新規ファイル / フォルダと並ぶ2項目
   - OS **File** メニュー → **知識フォルダのひな形** サブメニュー（最小 / 本っぽい章立て）
3. 対象親は、選択ディレクトリまたはワークスペースルート。
   OS メニューとサイドバー + はワークスペースルート配下に作成する。
4. 親直下に **新しい一意フォルダ**を1つ作り、その中に固定テンプレだけを書く。
   既存パスは上書きしない。衝突時は別名（`知識フォルダ 2` 等）または失敗。
5. 成功後、`index.md` を既存の Safe Editor tab で開き、ツリーを更新し、
   ステータスで「必要なら『知識フォルダ（OKF）を点検』で確認できます」と示す。
6. 作成後の自動フル点検は行わない（点検は従来どおり明示操作）。

English labels may use “Create knowledge folder starter (minimal)” /
“Create knowledge folder starter (book-like)”. UI は「OKF 正式対応」や
「準拠証明」を主張しない。副文として Draft 互換の出発点である旨を出す。

## Templates

固定テキストのみ。ネットワーク・AI・ユーザープロンプト展開なし。
本文の正本は `src/features/okf/scaffoldTemplates/assets/`（Vite `?raw`）。
Spec pin は review と共通（現在 `ee67a5c` / OKF v0.1 Draft）。pin 変更時は
assets と `analyzeOkfBundle` の整合をセットで直す（`docs/okf-spec-pin.md`）。

### A. `minimal`（starter）

既定フォルダ名: `知識フォルダ`

| 相対パス | 役割 |
|---|---|
| `index.md` | reserved index。`okf_version: "0.1"` と1件以上のリンク |
| `notes/first-note.md` | concept。`type: Note` |
| `log.md` | reserved log。1日分の `## YYYY-MM-DD` 例 |

### B. `book-like`（OKF 互換 book 雛形）

既定フォルダ名: `OKF互換の本`

| 相対パス | 役割 |
|---|---|
| `index.md` | ルート index + 章・ノートへのリンク |
| `chapters/01-morning.md` | `type: Chapter` |
| `chapters/02-afternoon.md` | `type: Chapter` |
| `notes/characters.md` | `type: Note` |
| `log.md` | reserved log 例 |

中身は smoke fixture `valid-root-versioned` と同系統の短い日本語プレースホルダでよい。
**Hazakura Book の章順 manifest ではない。** 作家が後から編集する Markdown 群。

## Write Boundary

- main Safe Editor window のみ。
- `workspace_root` 内の親パスに限定。
- フォルダ名に path separator / `..` を許さない。
- 各相対パスは相対のみ、`..` なし、深度・件数・総バイトに上限。
- 各ファイルは `create_new` 相当で作成。既存があれば全体を失敗させる
  （部分作成後は可能な範囲でクリーンアップ、または事前に全パス非存在を確認）。
- Markdown UTF-8 テキストのみ。実行・バイナリ・シンボリックリンク作成なし。
- Agent Workbench / Assist からは呼べない。

推奨実装: 1 Tauri command が「新ルート + ファイル群」をまとめて書く。
テンプレ本文の正本は `src/features/okf/scaffoldTemplates/assets/` 下の
Markdown 資産（Vite `?raw`）に置き、レジストリが相対パスへマップする。
文言や構成の差し替えは資産ファイル編集で行える。Rust はパス境界と原子的作成のみ。
ユーザー定義テンプレ UI は v1.12 対象外（レジストリ拡張で後から足せる）。

## Out Of Scope

- Book Scope / 章順 UI / multi-file を一冊として編集
- whole-book preview / EPUB export from multi-file
- 起動時・workspace open 時の自動 scaffold
- 既存フォルダへの「OKF 化」自動修復 / frontmatter 一括付与
- テンプレの高度カスタマイズ UI、ユーザー定義テンプレ保存
- 公式 OKF 認証や upstream `main` 追従
- v1.11 点検ロジックのセマンティクス変更

## Acceptance

| Slice | Acceptance |
|---|---|
| **T0 — Contract + templates** | 本設計、2テンプレ固定本文、spec pin、上書き禁止、日本語/英語/かな copy |
| **T1 — Bounded multi-file create** | workspace 内・main window・create_new・件数/バイト上限・失敗時に既存を壊さない。Rust テスト |
| **T2 — Safe Editor wiring** | Palette + フォルダ/ルート context menu。成功で `index.md` を開き、ツリー更新、点検を促す status。未開 workspace では無効 |
| **T3 — Smoke / docs** | 作成 → 点検で required が実質クリーン（または準備項目のみ）を fixture/smoke で確認。living docs と smoke-checklist を更新 |

## Verification

通常の source ゲート:

```bash
npm run build:vite
cargo fmt --manifest-path src-tauri/Cargo.toml -- --check
cargo test --manifest-path src-tauri/Cargo.toml
npm run build
git diff --check
```

手動: 空 workspace で2種のひな形作成 → `index.md` が開く → 点検が通る（または改善案のみ）。
既存 `index.md` がある親では **上書きせず** 新フォルダ名で作成されること。
