# v1.7 Trust And Daily Use Plan

Status: Active v1.7 hardening plan / review-derived
Scope: v1.7 Reference Compare plus Safe Editor trust, continuity, and long-document quality
Authority: High for the v1.7 follow-up queue
Last reviewed: 2026-07-11

## Decision

v1.7 should close the gap between source-landed capability and dependable daily use.
The active goal is not another large surface. It is:

> 原本を見ながらMarkdownを直し、モードを往復しても、長文でも、未保存でも失わない。

Reference Compare remains the v1.7 feature story:

- one editable Markdown buffer in the center / primary column;
- one read-only PDF, image, Markdown, or text reference on the right;
- Import Assist pairing and page-follow remain explicit and advisory;
- Reference Compare is not Diff, not two editable panes, and not a second document model.

The review-derived items below are planned v1.7 scope except **Book Scope Alpha**, which remains deferred to v2. Book Scope must not be pulled forward to satisfy this plan.

## Priority Slices

### P0 — trust before breadth

#### T-1: L Mode continuity

Make a presentation-layer toggle preserve editing continuity.

- Restore the side-pane state that was visible before entering L Mode.
- Do not silently discard Diff / compare selections; either restore them or explain the intentional reset.
- Preserve CodeMirror undo history across the L Mode toggle, or keep the toggle from remounting the editor through a proven safe mechanism.
- Keep cursor, selection, scroll ratio, IME behavior, Preview, Diff, export, and saved source semantics intact.

Acceptance:

- Preview or e-book → L Mode → normal returns to the previous surface without reconfiguration.
- An edit made before the toggle remains undoable after returning.
- Japanese IME, wrapped lines, and long-document cursor position remain stable.

#### T-2: explicit recovery for pathless drafts

Separate “no auto-save to the source file” from “no recovery after a crash.”

- Persist new-file and Import Assist drafts only in an app-private recovery store.
- Keep the record TTL- and size-bounded, revision-aware, and separate from the source file.
- Show recovery as an explicit candidate at startup; never merge or apply it silently.
- Remove the record after Save As, explicit discard, or confirmed cleanup.

Acceptance:

- Forced termination can recover an unsaved new/imported draft without changing its source file.
- A changed source file still requires explicit conflict/review handling.
- Storage failure is visible without interrupting normal typing.

#### T-3: Reference Compare packaged proof

Make the v1.7 feature usable in the real distribution shape before a version bump.

Minimum matrix:

- workspace and external Markdown/text reference;
- image reference;
- text-layer PDF and scanned PDF Import Assist pair;
- page follow, manual pause, resume, and page-level 要確認;
- Save As, tab switch, dirty close, reference replacement, source deletion;
- narrow window, keyboard-only operation, App Store sandbox picker, and PDF raster CSP.

Acceptance:

- The center remains the only editable/saveable document.
- Closing/replacing a reference never mutates the editor buffer.
- The packaged app proves the same boundary as unit tests and source docs.

### P1 — bounded scale and failure clarity

#### S-1: processing budgets

Add independent limits and visible fallback states for:

- synchronous draft persistence (`revision`, idle/debounce, per-document and total byte limits);
- Diff line dimensions, total bytes, output rows, and DOM/windowing;
- workspace search cumulative bytes, time, cancellation, and truncation reason;
- PDF export page/byte budgets and bounded merge memory;
- PDF/reference raster total memory and prefetch cancellation.

Do not introduce background indexing. A bounded, cancellable scan is sufficient for Safe Editor.

#### S-2: failure containment and regression signal

- Add a root-level error recovery surface that protects the editing session and offers a safe reload/diagnostic path.
- Make the PATH-sensitive Agent Workbench test deterministic; a full Rust test run must not appear green when an isolated rerun is the only passing evidence.
- Add integration coverage for `open → edit → Save As → Assist/Reference → close` state transitions.

Agent Workbench remains Developer / GitHub-only and does not become part of the Safe Editor flow.

#### S-3: long-reference and accessibility quality

- Virtualize or window long text references instead of creating one DOM node per line.
- Give workspace rename inputs an accessible name.
- Run a small keyboard, VoiceOver, contrast, enlarged text, and `prefers-reduced-motion` matrix across normal, L Mode, e-book, Reference, and modal states.
- Keep the existing accessible labels for separators, page controls, toggles, and dialogs.

#### S-4: discoverability without chrome expansion

- Add a small first-use explanation to Start Panel: write, read, verify.
- Explain the purpose of Preview / L Mode / e-book / Outline / Diff / Reference by task, not only by feature name.
- Show the e-book next-page affordance once, then return to the quiet presentation.
- Keep Assist understandable while off: on-device, availability-gated, explicit, and reviewable before save.

### P2 — bounded polish

- Combine EPUB/PDF metadata, image warnings, unsaved status, and output destination into a concise pre-export confirmation.
- Move Spellcheck into the main Preferences surface as well as Quick Settings.
- Measure theme shader work before adding more visual themes; preserve readability and reduced-motion behavior.
- Improve reference text readability and page-level advisory presentation after the long-reference path is bounded.

## Implementation Order

1. T-1 L Mode continuity and T-2 pathless recovery design/tests.
2. T-3 packaged Reference Compare proof and the v1.7 smoke matrix.
3. S-1 processing budgets, S-2 failure containment, and S-3 a11y / long-reference fixes.
4. S-4 discoverability and P2 export/theme polish.

Each slice should remain independently verifiable. Do not combine a data-safety fix with a visual redesign or a distribution-lane change.

## v1.7 Completion Gate

v1.7 is ready for version-bump discussion only when:

- Reference Compare packaged proof is complete for the matrix above;
- L Mode does not silently lose Undo or the prior reading/review context;
- unsaved new/imported drafts have explicit crash recovery;
- large Diff/search/reference/export paths have bounded failure states;
- keyboard/a11y smoke covers the new reference and mode transitions;
- the full Rust suite is deterministic, or any remaining failure is clearly isolated and explicitly accepted;
- README and release copy still describe v1.6 as the published version until the v1.7 bump is approved.

## Explicit Deferrals

Book Scope Alpha remains v2. It may introduce a small, explicit Markdown set, visible order, and reversible manifest only after the single-document and Reference Compare loop is proven. Do not add hidden workspace indexing, automatic chapter inference, a database-like book model, multi-file automatic rewriting, or Book Scope-wide AI behavior in v1.7.

Also out of v1.7:

- two editable panes or generic split-editor behavior;
- Review Desk revival;
- cloud OCR, generic AI chat, provider plugins, or background rewriting;
- Git, LSP, terminal, arbitrary command execution, or project-wide indexing;
- automatic save, auto-apply, auto-commit, or release automation inside the app;
- vertical writing as a new structural product lane.
