# v1.13 Interaction Clarity Plan

Status: Proposed after v1.12 review
Scope: Small v1.x UX slices that remove silent failure and avoidable ambiguity
Authority: Medium
Last reviewed: 2026-07-16

## Decision

The 2026-07-16 static UX review has a sound direction: keep one editing space
and make existing actions easier to understand. Do not copy its full 103-item
inventory into the active queue. v1.13 is a bounded clarity pass after the
submitted v1.12 lane; it is not a top-chrome redesign or a new product model.

The durable lens is:

> 操作後の無反応をなくし、対象・保持状態・未保存状態を明示する。

## Candidate Slices

Each row is independently selectable and should begin with a failing focused
test when practical. Recheck the live source before implementation because the
review was static and some observations are already covered by current code.

| Order | Slice | Current evidence | Acceptance boundary |
|---|---|---|---|
| 1 | **Command availability** | `Command` has no availability model; unavailable actions may rely on downstream guards. | Disabled commands expose a localized reason, cannot run with Enter, and remain readable by VoiceOver. No new command surface. |
| 2 | **Quick Open scope honesty** | Quick Open flattens the currently loaded workspace tree. | First ship accurate localized scope/truncation copy. A whole-workspace implementation requires a separate bounded Rust/path review; no background index. |
| 3 | **Search-open status localization** | Command Palette search result open still emits fixed English `Opened ...`. | English / Japanese / kana use one locale contract while diagnostic path/line detail remains intact. |
| 4 | **Quick Settings semantics** | The surface uses `role="menu"` while containing checkbox/range controls. | Use appropriate popover/dialog semantics, Tab navigation, Escape/outside-close rules, and focus return to the trigger. |
| 5 | **Preview / e-book loading feedback** | Lazy pane boundaries still use `Suspense fallback={null}`. | Avoid a blank-open impression with delayed, localized status; do not flash on fast loads or spam live regions. |
| 6 | **OKF scaffold progressive disclosure** | Basic file/folder creation and two OKF starters share the New surface. | Keep file/folder actions primary and group starters under a clearly named template/knowledge-folder section. Preserve all four explicit entry points and keyboard behavior. |
| 7 | **Workspace destructive-target clarity** | Footer Trash behavior depends on active/path/tree state and deserves interaction review. | Before action, expose the exact target and accurate disabled reason. Do not widen into a general file manager; confirmation and workspace containment remain unchanged. |

## Review Before Scheduling

These ideas fit v1.x, but need interaction evidence or a small design contract
before entering implementation order:

- same-name tab disambiguation and tab overflow;
- Reference retained-state wording and a shared right-pane header;
- recent workspaces and navigation history;
- purpose-led top controls (`書く・読む・確かめる・整える`);
- export result affordances and status-message lifetime rules.

Prefer one reproduced friction point and one focused slice. Do not batch these
into a shell rewrite.

## Already Covered Or Needs Revalidation

- Outline heading-level controls already have localized action labels and
  accessible names; do not schedule an arrow-label fix without a reproduced
  gap.
- Local Assist acceptance already states in its title copy that saving has not
  happened; improve visible wording only if hands-on review still shows
  ambiguity.
- Current purpose-led discovery copy already frames Preview / Reference /
  e-book / Outline / Diff. A new grouped top bar is optional redesign, not a
  P0 correction.

## Deferred Boundary

Keep these outside v1.13:

- whole-workspace Quick Open without explicit depth/count/time/symlink budgets;
- shared analysis caches that resemble a hidden document model;
- unified AI/OCR/Recovery candidate architecture;
- background indexing, startup workspace analysis, auto-apply, auto-save,
  OKF auto-repair, two editable panes, Git/LSP/terminal/plugin expansion.

## Verification Contract

For each selected slice:

1. Pin the reproduced ambiguity or silent-failure case with a focused failing
   test where practical.
2. Preserve Japanese / English / kana key parity and IME composition guards.
3. Check keyboard reachability, Escape behavior, focus restoration, and
   non-color state communication on the touched surface.
4. Run the relevant focused tests, `npm run typecheck`, `npm test`,
   `npm run build:vite`, and `git diff --check` when shared UI is changed.
5. Use packaged/manual smoke for WKWebView focus, VoiceOver speech, narrow
   windows, or other host-dependent claims; report unverified breadth plainly.

v1.12 App Review and publication remain a separate lane. Do not fold v1.13
source changes into the submitted candidate unless a reproduced hotfix blocker
requires an explicit decision.
