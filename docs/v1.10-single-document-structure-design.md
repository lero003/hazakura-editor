# v1.10 Single-document Structure Foundation

Status: Active implementation contract
Scope: One Markdown source; read-only structure first, bounded edits later
Authority: High for v1.10 implementation
Last reviewed: 2026-07-14

## Goal

一つの Markdown source にある frontmatter、ATX heading、明示的 page-break、
EPUB navigation候補を同じ解釈で扱う。Markdown source は保存の正本のままとし、
構造は再生成可能な presentation metadata とする。

## Invariants

- 編集可能な buffer は一つだけ。
- source offset / line を保持し、解析のために本文を正規化・再構成しない。
- fenced code、leading YAML frontmatter、LF / CRLF を同じ規則で扱う。
- 空見出しや level skip は将来の助言候補であり、解析エラーや自動修正にしない。
- structure edit は後続スライスでも明示操作、単一 transaction、Undo可能、dirtyの対象。
- manifest、独自database、workspace scan、複数ファイル章順、auto-saveを追加しない。

## Slices

| Slice | Status | Acceptance |
|---|---|---|
| **S1 — Shared read-only model** | **Done / source** | `parseMarkdownStructure` が frontmatter、ATX heading、page-break、navigation候補を source位置付きで返す。Outline、e-book、EPUBが同じ解析を使い、異なる長さ/種類の fence と CRLF を回帰テストで固定する。 |
| **S2 — Structure overview** | Next | 既存 Outline surface 内で hierarchy と page-break を読み取り専用表示し、選択でEditorへ移動する。新しい pane / chromeは作らない。 |
| **S3 — Advisory checks** | Later | skipped level、空見出し、重複navigation label、極端に長いsectionを助言として示す。保存やexportを妨げない。 |
| **S4 — One bounded edit proof** | Later | heading level change またはsection moveの片方だけを、明示的な単一-buffer transactionとして実証する。Undo、dirty、Save As、recovery、IMEを確認する。 |

## S1 Model

`src/features/editor/markdownStructure.ts` は次だけを返す。

- closed leading YAML frontmatter range
- ATX heading level、line、source offsets、表示text、navigation label候補
- standalone page-break markerのsource offsetsと render / drop role

空見出しは heading として残すが navigation候補にはしない。これにより後続の助言は
構造を見失わず、EPUB navigationは空labelを生成しない。
