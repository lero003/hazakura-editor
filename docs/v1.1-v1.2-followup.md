# v1.1-v1.2 Follow-up

Status: Operational
Scope: User-observed follow-up after the v1.0 Golden Manuscript smoke
Authority: High
Last reviewed: 2026-06-30 (v1.1.0 released; v1.2 Polish lane open)

## Decision

The 2026-06-28 user-side pre-v1 checklist supports a v1.0 Go decision.
The checklist is evidence, not a requirement that every box be checked.
The decisive No-Go conditions were not observed: the app launches,
source was not lost, long-form e-book reading remained usable, EPUB
export and page breaks worked in Apple Books, Local Assist applied to the
intended document through explicit review, and the App Store lane kept
its safety boundary.

`1.0.0` was subsequently approved and released on the Mac App Store.
`1.1.0` followed with the Reader / Editor / Preview position-continuity
slice and has also been approved and released. No hotfix blocker has
been reported for either version. The canonical listing packet remains
`docs/releases/1.0.0-app-store-listing-copy.md`; the published artifacts
are immutable, and the observations below now form the active v1.2
Polish intake queue.

## v1.1 Kickoff (closed)

`1.1.0` shipped the **Reader / editor position ownership** and
**Preview / tab position persistence** continuity work. The intended
mental model held:

- explicit entry from the editor or Preview uses the current visible
  source position;
- a passive reader remount preserves the active reader location;
- `編集に戻る` returns through the active reader source estimate;
- Preview reopen and tab/external-link transitions do not silently reset
  an unrelated editor position.

The v1.1 candidate lane is closed. Image-heavy layout, Diff wording,
menu containment, and backup-file investigation now ride the v1.2 Polish
queue below, one case at a time.

## v1.1 Position Continuity: Current Behavior Pins

The five transitions were reproduced independently before the shared
ownership change. Completion state as of 2026-06-29:

1. **Explicit e-book entry** (editor -> e-book Mode). Pinned: when the
   side pane switches to e-book Mode and a visible editor scroll heading
   exists, the reader opens at that heading's chapter. Already covered
   by `AppWorkspace.test.tsx` "opens the e-book pane near the current
   editor heading" and "uses the visible editor scroll heading".
   Matches the intended mental model; no change needed.
2. **Passive reader remount.** Fixed (#2) 2026-06-29: `ebookLocation` was a
   single slot keyed by `documentKey` (so A -> B move -> A lost A's reader
   position). It is now a per-`documentKey` Map, so each tab keeps its own
   reader location and returning to a tab restores it.
   `AppWorkspace.test.tsx` verifies A -> B move -> A restores A's location.
3. **`編集に戻る` (return to edit).** Pinned: closes Reading Focus and
   moves the editor to the reader source estimate (`sourceLine` if
   available, else the chapter heading), with editor focus. Already
   covered by "returns from Reading Focus to the active reader chapter
   heading" and "...to an estimated reader source line". Matches the
   intended mental model; no change needed.
4. **Preview reopen (top-reset).** Reproduced in the built app. Replacing
   rendered HTML collapses the shared scroll container during the existing
   debounce, so editor-to-Preview sync can run before the new layout exists.
   `PreviewPane` now reports render completion and `SidePane` restores the
   current document's ratio afterward. README -> AGENTS -> README and
   Preview -> e-book -> Preview both restored the prior area.
5. **Tab / safe local-link transitions.** `EditorPane` now restores the
   registry selection and scroll after each document mount, with selection
   offsets clamped to current text length. A built-app local Markdown-link
   transition to the documentation index and back preserved the source and
   Preview areas. Link safety/opening behavior itself is unchanged.

`AppWorkspace` owns one `DocumentViewStateRegistry`, with nested Editor,
e-book, and Preview state patched independently so one surface cannot erase
another. Focused tests cover state preservation, stale-document guards,
offset clamping, A -> B -> A restoration, and post-render Preview restore.

## v1.1: Daily-use Continuity And Trust

Prioritize issues that can make a writer lose position, confidence, or
the meaning of a review action.

1. **Reader / editor position ownership.** After e-book Mode has moved,
   reopening it may prefer stored reader position over the current editor
   position. Define an understandable rule for explicit editor entry,
   stored reader continuation, and `編集に戻る`.
2. **Preview and tab position persistence.** Preview can reopen at the
   top, and external-link or tab transitions may reset the editor
   position. Reproduce each transition independently before changing the
   shared position bridge.
3. **Recovery availability after forced termination.** Reproduce with a
   local folder and a Google Drive folder separately. Record whether the
   backup had time to be written, whether the workspace bookmark was
   restored, and whether reopening the same document suppresses the
   candidate. No source loss was observed, so this is a focused
   reliability investigation rather than a v1 blocker.
4. **Image-heavy layout edge cases.** Cover consecutive images in e-book
   Mode, a first-line or nested-path image in L Mode, and PDF page breaks
   through a large image. Preserve the passing ordinary image/text case.
5. **Diff action clarity.** Evaluate changing the floating comparison
   actions from a close-like affordance to explicit `採用` / `破棄`, and
   keep dirty/save state visible when the file-comparison surface is open.
6. **Context-menu viewport containment.** Reposition or flip the
   right-click menu when it would extend below the window.
7. **Unexpected backup files in Trash.** Determine whether recurring
   `.bak` files come from Hazakura, a cloud-sync interaction, or another
   agent/machine before changing backup cleanup behavior.

## v1.2: Polish And Expectation Setting

`1.1.0` was approved and released on the Mac App Store. The v1.2 Polish
lane now owns the items below. Items marked `Done` were verified during
the v1.2 planning pass and need no code change; items marked
`Implemented` shipped in the v1.2 source lane; items marked
`Observation` stay gated on user data.

1. **Local Assist quality.** `Observation.` Collect short examples of
   unclear output, then prefer one small prompt adjustment at a time.
   Keep `プレビュー`, output variability, explicit Diff review, and
   no-auto-save wording. The per-action request text is duplicated by
   design across `src/lib/appleAssist/instruction.ts` (UI display /
   fallback) and
   `src-helpers/apple-assist/Sources/HazakuraAppleAssist/GenerateCandidate.swift`
   (`requestTemplate`, the actual on-device LLM fallback) because they
   sit on opposite sides of the helper process boundary; any wording
   change must update both sides in lockstep. No wording change is
   scheduled until a concrete unclear-output example arrives.
2. **Export UX.** `Implemented` (scope note), `Done` (button order).
   The EPUB settings dialog button order was already aligned with the
   other in-app dialogs (`[主アクション] [Cancel]`, shared
   `.dialog-actions`); the earlier note was stale. The dialog now also
   carries an in-app scope note stating that e-book Mode is a reading
   preview, the exported EPUB targets a single Markdown document, and
   Hazakura is not a full EPUB production tool (Japanese and English).
   See `src/components/app/EpubExportSettingsDialog.tsx`.
3. **HTML image-size limit.** `Observation.` The limit is a three-layer
   policy, not one number: 10 MB `MAX_EDITABLE_BYTES` (text-open /
   edit cap), 20 MB `MAX_IMAGE_PREVIEW_BYTES` (workspace image preview
   / paste / drag-import), and 2 MB `MAX_EMBEDDED_IMAGE_BYTES` (in-WebView
   embedded data URL during render / export). The binary sniff
   (`looks_binary`) and paste memory guard (`decoded_base64_len`) stay
   in place. Raising any layer needs image-heavy fixture evaluation
   first; do not raise on assumption.
4. **L Mode task checkboxes.** `Done.` Direct checkbox toggling is
   already implemented in both interaction modes: click
   (`lModeTaskClickViewPlugin` click handler) and keyboard (Enter/Space
   with IME-composition guards) in
   `src/features/editor/lMode/taskWidget.ts`. Source-first display and
   checkbox toggling coexist by design: the active (cursor) line keeps
   the raw `[ ]` source text editable, while inactive lines render the
   `LModeTaskWidget`, which dispatches the exact same 3-char replace so
   source and view stay byte-identical. No rule change needed.
5. **Command discovery.** `Implemented.` The command palette
   (`Cmd+Shift+P`, `useCommandPaletteController.ts`) now exposes the
   same safe Markdown actions as the right-click slash menu — headings,
   bullet / numbered / task lists, quote, code block, divider, image,
   strikethrough, plus the existing bold / italic / inline-code / link /
   3-column table / date / time actions. Palette execution is derived from
   the existing slash-command registry, so insert text and action callbacks
   stay shared; table insertion preserves a selected range instead of
   replacing it. Palette labels stay English to match the existing palette
   copy; full palette i18n is a separate future slice.
6. **Empty-state composition.** `Observation.` `StartPanel.tsx` keeps the
   current logo placement. The v1 screenshot set
   (`docs/releases/1.0.0-app-store-listing-copy.md`) leads with the
   editing surface, not the empty state, so no concrete imbalance has
   surfaced. Revisit only when a screenshot exposes one.
7. **Experimental workspace drag/drop.** `Deferred (user direction).`
   Held out of the v1.2 scope by user direction. The bounded
   workspace-internal drag/drop Move stays de-emphasized (no context-menu
   entry, drag/drop only); New File, New Folder, Rename, and Move to
   Trash remain the dependable file-tree operations. Do not broaden into
   a full file manager.

## Accepted Limits And Optional Proof

- 100k-character documents are not a comfort guarantee. The priority is
  no crash, source loss, or silent failure; measure before optimizing.
- Save As may reset scroll position and undo history until the editor
  session id is separated from the document key.
- PDF margin and page-size controls, advanced EPUB metadata, cover and
  navigation editing, multi-file books, and vertical writing remain
  post-v1 work.
- Apple Books proof is sufficient for v1. Kindle Previewer remains an
  optional compatibility check, not a release gate.
- Local Assist remains a preview companion, not a strong general-purpose
  AI editor.

## Intake Rule

Take one reproducible item at a time. Record the document shape,
workspace type, exact transition, expected result, and observed result.
Do not bundle unrelated visual cleanup or refactoring with a fix. A
verified no-op or accepted limitation is a valid outcome.
