# v1.xx Image / Media Boundary Plan

Status: Active plan (M0–M4 Keep in source; packaged hands-on smoke pending)
Scope: Safer loosening of Markdown image preview and export without becoming a browser or file manager
Authority: Medium — product direction for v1.xx; implementation needs focused slices and security-boundary updates
Last reviewed: 2026-07-17 (M0–M3 implementation)

## Decision

Hazakura は既定では **外部画像を勝手に読まない**（`docs/security-boundary.md`）。
一方、実原稿では次が頻繁に起きる。

- workspace の切り方が浅く、文書相対の `../assets/...` が root 外になる
- 他フォルダや一時パスの図を参照したい
- 一般 Markdown と同様に `https://…` の図を一時的に見たい
- 書き出し物はオフラインで自己完結させたい

したがって v1.xx では、境界を捨てるのではなく **同意・設定・書き出し時の実体化**で
読み込みを段階的にゆるくする。版番号は固定しない。try-and-error の Theme
（Media / Trust edges）として `docs/v1.13-plus-refinement-roadmap.md` から選ぶ。

Durable lens:

> 既定は安全。表示のゆるさは同意と設定で足す。書き出しは自己完結を優先する。
> source は黙って書き換えない。

## Current Baseline (do not regress)

As of `1.12.0` source:

| Source kind | Preview / e-book | Notes |
|---|---|---|
| Relative path inside selected workspace | Allowed | Document-relative resolve → size-bounded load → data URL. Not limited to `assets/`. |
| Path that escapes workspace | Blocked | Message suggests opening a parent folder as workspace. |
| Absolute path outside workspace | Blocked | No arbitrary disk open. |
| `http:` / `https:` image URL | Blocked | No automatic external fetch. |
| Bounded `data:image` (png/jpeg/gif/webp) | Allowed | SVG and oversized payloads blocked. |

Export (HTML / PDF / EPUB) already packages **workspace-resolved** images where
possible and warns on unresolved ones. Drag/paste can place files under the
selected workspace `assets/` and insert a Markdown reference.

Pain is mostly **workspace cut too deep**, **outside-local**, and **remote URL**,
not “must live only under `assets/`”.

## Goals

1. Reduce false “画像を表示できません” for legitimate manuscript layouts.
2. Allow outside-workspace local images and remote images **only with user control**.
3. Keep export packages offline-safe when the user chooses to materialize.
4. Expose clear Preferences / per-document controls so power users can opt in
   without making the default leaky.
5. Preserve Safe Editor identity: no silent network, no silent source rewrite,
   no unrestricted filesystem.

## Non-Goals

- Becoming a browser (arbitrary HTML, tracking, cookies, auth’d CDN sessions).
- Project-wide image indexing or startup fetch of all Markdown images.
- Auto-rewriting Markdown paths without an explicit, undoable user action.
- Opening arbitrary absolute paths with no picker / bookmark / containment story.
- Changing OKF review to render bundle images.
- Import Assist extracting binary images from PDF (separate lane if ever revisited).
- v2 multi-file Book asset graph as a hidden model.

## Control Surfaces

Three layers of user control. Prefer the least global option that solves the task.

### 1. Defaults (always on)

- Workspace-internal relative images: load within existing size/type budgets.
- Everything else: show a blocked note with a **specific reason** and next actions
  (open parent workspace, allow this path, enable remote for this document,
  copy into `assets/`, open Preferences).

### 2. Preferences (sticky, explicit)

Settings candidates (names are planning labels):

| Preference | Default | Effect |
|---|---|---|
| Load remote images in Preview / e-book | **Off** | When On, `http(s)` images may load after network policy checks. Still no auto-export fetch without export consent. |
| Allow images outside workspace (with prompt) | **Off** or **Ask** | Off = always block. Ask = per-path or per-folder prompt. On = session/document still needs a containment model (bookmark / previously approved roots). |
| Prefer packaging images on export | **On** (recommended) | Export materializes resolvable images into the package. |
| Offer “copy into assets and rewrite path” after allow | Off or On | Only as an explicit post-action; rewrite is always undoable/dirty. |

Preferences must:

- use natural Japanese / English / kana copy;
- state privacy cost of remote load (server may see a request);
- never imply “safe browsing” or encryption;
- be discoverable from the blocked-image note and from Settings under a media /
  privacy-adjacent group — not a giant settings redesign.

### 3. Session / document consent (narrower than Preferences)

Even when a Preference is On or Ask:

- first remote load in a document may show a one-shot confirmation if product
  review wants belt-and-suspenders;
- outside-workspace paths should record **approved path or parent folder**, not
  “whole disk”;
- consent is revocable (status / Settings / “この文書の画像許可を解除”);
- Assist, Agent Workbench, OKF scan, and background jobs never inherit image
  fetch permission as a side channel.

## Technical Boundary Shape

### Preview / e-book / HTML render path

```text
Markdown img src
  ├─ allowed data:image  → keep (existing)
  ├─ workspace-local     → load (existing)
  ├─ outside-local       → block | prompt | load if approved root
  └─ http(s)             → block | load only if Preference (+ optional confirm)
```

Hard rules:

- Cap decode size / dimensions (reuse ~20 MB local and inline caps as baselines).
- Reject `data:image/svg+xml`, active content, and non-image MIME pretenders.
- Remote: timeout, max bytes, no redirects, no cookie jar design.
- Never navigate the app WebView to the image URL; only bounded fetch → data URL
  or blocked note.
- CSP / Tauri capabilities updated in the same slice as any network image path.
- Status or note must say whether an image came from workspace, approved local,
  or remote.

### Export (HTML / PDF / EPUB)

Separate from Preview:

| Step | Behavior |
|---|---|
| Resolve | Same path rules as Preview for local; remote only if export materialize is requested / Preference allows export fetch. |
| Materialize | Copy bytes into package (`OEBPS/images/…` or HTML sidecar assets). Prefer content-hash or stable names; no overwrite of unrelated files. |
| Fail soft | Unresolved / oversize / denied → visible warning list; do not abort entire export unless user chose strict mode later. |
| Source rewrite | **Default off.** Optional explicit “assets に固定して Markdown を更新” creates local copies under workspace `assets/` (or document-adjacent folder), rewrites references in the active buffer as one Undo-able transaction, marks dirty. Never auto-save. |

Recommended mental model:

- **Preview** may be temporarily wide (consent).
- **Export package** should be self-contained when materialize succeeds.
- **Markdown source** stays the user’s file until they opt into rewrite.

### Outside-workspace local access

Do not implement as “any absolute path string works.”

Preferred order:

1. User picks file/folder via OS dialog → security-scoped access where required.
2. Or user approves a concrete resolved path from a blocked note.
3. Record approval with scope (file vs parent directory) and lifetime
   (session / until revoke / remember for this workspace pair — product choice
   in the implementation slice).
4. Symlink escape and `..` outside approved scope still refuse.

### Remote access

- Preference default **Off**.
- Schemes: `https` preferred; `http` either blocked or same Preference with
  clearer warning.
- No remote images in OKF review HTML (review stays metadata-only).
- No remote images for Assist / Agent surfaces unless a future boundary review
  says otherwise (default: no).

## Experiment Sequence (v1.xx, not fixed versions)

Ship via the refinement loop: one hypothesis per run. Labels below are shipping
boxes, not commitments.

### Slice M0 — Honesty without new power (low risk) — **Keep (source)**

- Improve blocked-image copy: path/reference, reason, next actions.
- Stable keys: `data-hazakura-image-block` (`outside-workspace` | `absolute-outside` |
  `remote` | `unsupported-scheme` | `unsafe-data` | `invalid-src` | `missing-context` |
  `load-failed`). Preview / e-book / HTML share `classifyMarkdownImageSource`;
  load-failed notes also cover PDF embed failures.
- Next-action **text** (not live command buttons yet): parent workspace, assets
  relative path, local save for remote. Preference deep-links deferred to M1/M2.
- Keep all load policies identical.
- Acceptance: no new network or outside-root reads; tests pin reason keys + copy.

### Slice M1 — Outside-local with consent (medium risk) — **Keep (source)**

- Preference: ワークスペース外の画像 = **off / ask (default) / remember**.
- Preview / e-book blocked notes offer **親フォルダを許可して表示** when ask/remember.
- Approved roots are session-shared; remember persists per workspace root.
- Load via `open_local_image_under_roots` (canonical path under approved roots only).
- Acceptance: unapproved path still blocked; empty roots reject; no source rewrite.

### Slice M2 — Remote Preference (higher risk) — **Keep (source)**

- Settings: “Preview でリモート画像を読み込む” **default Off**.
- When On, Preview / e-book may fetch bounded **https only** via Rust `fetch_remote_image`
  (timeout, size, magic-bytes MIME). **http always blocked.**
- Blocked note when Off still shows host/path safely (no live request).
- Privacy sentence in Preferences hint.
- Acceptance: default Off; http rejected in Rust tests; no CSP `img-src https:` needed
  (data URL after fetch).

### Slice M3 — Export materialize (medium risk, high value) — **Keep (source)**

- Preference: “書き出しに画像を含める” **default On**.
- HTML / PDF / EPUB export uses same mediaAccess + loaders when materialize is On
  (workspace + approved-local; remote only if remote Preference also On).
- Failures remain soft (blocked notes / warnings). Source not rewritten.
- Acceptance: export path unit tests; no silent network without both prefs.

### Slice M4 — Explicit asset pin + rewrite — **Keep (source + palette)**

- Pure helpers: `pinExternalImages.ts` / `pinExternalImagesToAssets.ts`.
- Disk pin reuses `import_image_from_path` / `save_pasted_image` / remote fetch.
- When Preview remote loading is Off, an explicit warning confirmation is required
  before pin may contact any `https` image host.
- Command Palette: `file.pinExternalImages`
  （「外部画像を assets に固定…」 / en / kana）.
- One CodeMirror `replaceDocumentContents` transaction → dirty + one Undo.
- Refuses Assist-lock and IME composition; never auto-save.
- Acceptance: pin action tests + palette disabled reasons (need doc / workspace).

### Suggested shipping grouping

| Keep results | Possible box | Notes |
|---|---|---|
| M0 (+ maybe part of clarity) | early v1.13.x | Safe anytime after v1.12 review. |
| M1 | v1.1x media lane | Solves parent-folder friction without network. |
| M2 + M3 | later v1.1x | Remote is incomplete without export story. |
| M4 | v1.13 candidate | Kept after boundary review; packaged hands-on smoke remains. |

Jump order is allowed: if export pain dominates, M3 design can start after M1
even before remote Preference ships (materialize local-only first).

## Settings UX Notes

- Group under something like **表示とメディア** / **プライバシーと画像**, not
  buried only in developer text.
- Master switches:
  - リモート画像を Preview で読み込む（既定オフ）
  - ワークスペース外の画像（オフ / 都度確認 / 許可済みのみ）
- Secondary:
  - 書き出し時に取得可能な画像をパッケージへ含める（既定オン推奨）
  - 許可のクリア / この文書の一時許可を解除
- Quick Settings may surface a compact “この文書でリモート画像を許可” only if
  it does not turn Quick Settings into a second Preferences app.

## Open Questions (resolved / remaining)

1. Outside-local approval lifetime: **ask = session; remember = per-workspace localStorage.**
2. `http`: **always blocked**; https only when Preference On.
3. Export strict mode (“fail if any image missing”): **deferred**.
4. e-book Reading Focus: **same Preference / mediaAccess as Preview.**
5. Path display: **truncated ref in note; full path only on data attribute for approve.**
