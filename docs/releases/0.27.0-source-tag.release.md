# Hazakura Editor 0.27.0 Source / Local App Tag

> **continuous safe editing space refinement**

Status: Prepared for a GitHub source / local-app tag on 2026-06-20. No
binary assets are attached by this note.

`v0.27.0` is the source / local-app checkpoint for the v0.27 refinement
slice: large-document startup stability, mode-context continuity,
flow-preserving heading jumps, status-bar structure cleanup, and a
focused editor-scrollbar fix. Markdown source remains canonical, and the
release stays inside the existing Safe Editor / Preview / e-book / Diff /
Review surfaces.

This release publishes the repository state, source archive, release
notes, and build-from-source path. It does not attach a DMG asset. The
latest downloadable warning-expected DMG preview remains `v0.20.0`.

The App Store lane remains helper-free and omits Agent Workbench, CLI
Agent launch, Apple Local Assist helper, external AI/API calls, and
arbitrary command execution surfaces.

This is not a Developer ID signed, notarized, updater, or
installer-grade distribution.

## What Changed Since 0.26.0

### Large-document Startup Stability

- Preview rendering is deferred out of the initial React render path, so
  large restored Markdown documents no longer make Preview compete with
  the editor's first commit.
- Preview clears stale content before paint, shows a theme-bound loading
  surface, and then runs the existing Markdown render / workspace-image
  inlining path.
- Workspace restore now shows a theme-bound loading surface while
  persisted files are opening, instead of briefly falling back to the
  start panel.

### One Editing Space Context

- e-book Mode remembers the active chapter / page for the active
  document when switching away to Preview / another pane and back.
- This keeps Preview and e-book feeling like layers over the same
  Markdown source without adding persistence, indexing, or a second
  document model.

### Flow-preserving Navigation

- Outline / heading jumps now report the settled editor pixel scroll
  ratio after the CodeMirror jump, reducing the mismatch caused by using
  line-number ratio as a Preview scroll ratio.
- A follow-up editor-scrollbar fix blurs the CodeMirror editing DOM only
  when the native scrollbar gutter is pressed, avoiding the observed
  "second large manual scroll returns to the old focus area" behavior.

### Status Bar And Right-pane Controls

- Normal Safe Editor status now splits active document metadata into
  primary detail and secondary detail instead of relying on one compacted
  `statusDetail` string.
- Normal mode keeps LF / CRLF and encoding selectors reachable while
  moving lower-priority metadata into hover/title detail.
- L Mode keeps format controls hidden and shows the combined quiet
  detail, preserving its focused-writing surface.
- The L Mode top chrome no longer shows the Developer build badge.
- Review / Diff / Outline are exposed as independent right-pane controls
  instead of being collapsed into one menu.

### Small Post-0.26 Polish Included In This Tag

- Generic right-bottom error/recovery banners can be dismissed when they
  have no explicit recovery action.
- e-book page-break markers no longer render as both a horizontal rule
  and a page-break cue.

## Release Boundary

v0.27.0 does not add:

- A new GitHub DMG asset.
- Developer ID notarization, stapling, auto-update, or installer-grade
  distribution.
- AI Markdown ingest, automatic agent application, auto-save of
  candidates, or a new Review Desk model.
- Workspace As Book, project-wide indexing, multi-file book ordering, or
  a separate EPUB document model.
- Native Vibrancy Phase 2 beyond the already-shipped macOS 26 floor.
- Git, LSP, terminal, plugin, arbitrary command execution, or a provider
  marketplace.

## Known Limitations

- Boot-time theme pre-application remains a deferred improvement. A
  React-before-paint boot-theme attempt was technically plausible but
  made CodeMirror / Tauri first-paint layout failures difficult to
  isolate in the real app, so it is intentionally held for a separate
  debug slice.
- Very long-document editor / Preview scroll sync can still drift
  somewhat because the two surfaces do not share a semantic source map.
  The v0.27 fix improves jump feedback but does not build a source-aware
  Preview synchronizer.
- The latest downloadable warning-expected DMG preview remains
  `v0.20.0`; users who need a binary preview should use that release
  until a later DMG preview is published.
