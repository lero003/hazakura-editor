## App Store Connect Copy (draft)

Hazakura Editor v1.14は、日々の執筆で「今どこにいるか」「何が起きたか」が
分かりやすくなるよう、表示と案内を整えたリリースです。

・同名の開いているファイルは、親フォルダ名で見分けやすくしました。
・PDF / HTML / EPUB 書き出し成功時に、保存先パスと画像の警告をステータスに残します。
・Hazakura Local Assist の生成中ロック理由と、「採用してもまだ保存されていない」案内を明確にしました。
・参照を隠しても読み込みが残っているときは、参照トグル上で保持状態が分かるようにしました。
・スタート画面から、最近開いたフォルダを上限付きで明示選択して開き直せます（自動スキャンなし）。
・プレビューと電子書籍の違いを短く示し、電子書籍から「この位置を編集」で原稿位置へ戻れます。
・右ペイン（プレビュー / 電子書籍 / アウトライン / 差分 / 参照）に共通ヘッダーを置き、閉じる操作をそろえました。
・PDF参照の150%表示で、マウスホイールによる縦横スクロールを操作しやすくしました。
・アウトラインの構造助言をヒント表現にし、見出しレベルの変更は Cmd+Z で戻せる旨を伝えます。
・知識フォルダのひな形は作成前に置くファイルを説明し、点検では「まずここを直す」案内を出します（自動修正なし）。
・Import Assist の結果は未保存の下書きであること、保存するまでディスクに書かないことを明示します。

Markdown sourceを正本とし、外部AIへのnetwork fallback、auto-save、tool calling、workspace-wide indexingを行わない境界は変わりません。

## Boundary

- Safe Editor remains primary; Markdown/text source remains canonical.
- No Book Scope, background indexing, auto-apply, auto-save, or second editable buffer.
- App Store lane omits Agent Workbench, CLI Agent, arbitrary command execution,
  and external AI/API providers.
- Hazakura Local Assist remains on-device, explicit, unsaved, and reviewable.
- OKF review and scaffold stay explicit, bounded, and non-repairing.

## Explicit Non-goals

- multi-file Book Scope, chapter ordering, whole-book export
- vertical writing (縦書き)
- whole-workspace Quick Open / background index
- Git, LSP, terminal, plugins, provider-add UI
