# Hazakura Editor 0.32.0 App Store / TestFlight Candidate

> **Helper-enabled App Store lane package candidate**

Status: local package evidence generated on 2026-06-23 for TestFlight-side
v0.32 Editor / Reader Position Bridge smoke. App Store Connect upload,
processing, TestFlight install / launch, App Review, and release handling
are performed outside this repository unless separately recorded.

`0.32.0` packages the v0.32 reader/editor bridge work after the local
`0.31.0` Reading Focus / Spread View candidate. The release boundary
stays unchanged: Markdown source remains canonical, Safe Editor remains
primary, and the App Store lane omits Agent Workbench, CLI Agent launch,
dev mode, arbitrary command execution, external AI/API calls, Git, LSP,
terminal, plugins, workspace-wide indexing, auto-apply, and auto-commit
behavior. Hazakura Local Assist remains only an explicit, on-device
writing companion whose final result enters the existing unsaved edit
transaction / Diff review flow.

## v0.32 Candidate Focus

- Opening e-book Mode anchors near the current editor cursor or visible
  scroll position when no fresher reader location exists.
- Reading Focus can return to an approximate Markdown source line through
  the existing editor jump path.
- Right-pane one-page reader navigation can move the editor to the same
  chapter area without entering Reading Focus first.
- Side-pane one-page reading and Reading Focus spread reading share the
  same reader location state.
- Passive source edits and chapter reclassification do not push the
  editor position back from the reader.
- Reading Focus spread view can use a spare right page at a chapter
  boundary to preview the next chapter while keeping reader/editor
  location anchored to the active chapter.
