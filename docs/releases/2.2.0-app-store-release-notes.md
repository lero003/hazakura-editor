# Hazakura Editor 2.2.0 App Store Release Notes

Status: Local candidate preparation; not uploaded or published
Scope: User-facing App Store / TestFlight release notes for package `2.2.0`
Authority: Medium
Last reviewed: 2026-07-22

This note covers the v2.2 quality pack that builds on v2.1 (whole-book Reader
search + Preview image-loading hardening). Published Mac App Store version and
immutable source tag remain `2.0.0` / `v2.0.0` until a separate human upload
and review. Local package provenance lives in ignored
`docs/internal/app-store-candidates/latest.json`.

## App Store Connect Copy

Hazakura Editor v2.2では、本としての読み書きまわりの品質を整えました。

・「本全体を読む」で、いま読んでいる章の表示と前の章・次の章への移動ができます。
・本の空状態で、章の順序がこのMacのアプリ内に保存されること（フォルダ並びやOKFの順序ではないこと）を案内します。
・PDF / EPUB / HTML の書き出しが終わると、完成したファイルをFinderで表示します。
・本全体の書き出しでは、準備・章の読み込み・書き込みの各段階をステータスで伝えます。
・書き出し前の点検メッセージに、次に直す手がかりを付けました。
・Hazakura Local Assistが使えないとき、理由をボタンと設定に素直に表示します。
・v2.1の本の中検索と、画像が多いPreviewの遅延読込はそのまま引き継ぎます。

Markdown sourceを正本とし、background indexing、auto-save、auto-apply、
workspace-wide scan、縦書きを行わない境界は変わりません。

## Candidate Boundary

- Reader navigation only reorders scroll within the already loaded whole-book
  budget (100 chapters / 32 MiB). It does not add a second editable buffer or a
  persistent reading-position database as a product claim beyond the current
  session indicator.
- Book order remains app-private settings (not a project file). Empty-state and
  Help copy keep that disclosure.
- Export reveal uses the existing `reveal_path_in_file_manager` command after a
  successful save. Failures to reveal do not undo a completed export.
- Local Assist honesty is presentation-only: probe results already exist; this
  release surfaces them in chrome and Preferences without adding static lint,
  network fallback, or auto-apply.
- Vertical writing (縦書き) remains deferred after the horizontal Book foundation.
- App Store lane still omits Agent Workbench, CLI Agent, arbitrary command
  execution, and external AI/API providers.

## Includes From v2.1 (still in package)

- Bounded whole-book Reader search over loaded chapter names and visible
  Markdown, including unsaved live buffers.
- Interactive Preview / whole-book Reader near-viewport image loading with a
  two-read concurrency cap per pane.
