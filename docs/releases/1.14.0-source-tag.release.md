# Hazakura Editor 1.14.0 Source / Local App Release

> **書く・読む・確かめるための継続性と状態案内を改善**

Status: Published as a GitHub source / local-app tag on 2026-07-18. No binary
assets are attached.

`v1.14.0` is the source / local-app checkpoint for the post-v1.13 refinement
Keep set. It publishes the repository state, source archive, release notes, and
build-from-source path. It does not attach an App Store package or DMG.

The same `1.14.0` product line has a selected local App Store submission
candidate. The source tag does not prove App Store Connect upload, TestFlight
processing, App Review, or publication; those remain human-side gates.

## Build From Source

```bash
npm ci
npm run build
```

The default local smoke bundle is generated at:

```txt
src-tauri/target/release/bundle/macos/Hazakura Editor.app
```

It is a local validation artifact, not an attached, notarized, or App
Store-distributed binary.

## What Changed Since 1.12.0

- Same-name tabs show parent folders; Reference retention and shared right-pane
  headers make the current reading context easier to recognize.
- The returning Start Panel is compact, keeps basic actions near the top, and
  exposes recent folders without background scanning.
- PDF Reference 150% uses one overflow owner and improves standard-mouse wheel
  handoff while preserving native two-axis trackpad input.
- Export success keeps destination and image-warning context visible.
- Local Assist and Import Assist make generated, adopted, and unsaved states
  explicit without auto-save or network fallback.
- Preview / e-book / Outline guidance clarifies reading versus editing and keeps
  heading changes Undo-able.
- OKF scaffold and review copy explain files and the first manual fix without
  adding automatic repair, indexing, or Book Scope.

## Release Boundary

v1.14.0 does not add:

- multi-file Book Scope, chapter ordering, or vertical writing
- background indexing, automatic repair, auto-apply, or a second editable buffer
- Agent Workbench / external CLI agents in the App Store lane
- network fallback, auto-save, tool calling, Git, LSP, terminal, or plugins
- binary assets on the GitHub tag

Markdown / text source remains canonical. Safe Editor remains primary.

## Known Limits

- App Store Connect upload, processing, TestFlight, review, and publication are
  not release claims of this source tag.
- Spoken VoiceOver, narrow-window, long-document, IME, and full signed
  TestFlight breadth remain hands-on evidence.
- Optional residual polish (tab overflow, navigation history, and global status
  lifetime) remains follow-up rather than a blocker for this checkpoint.
- `cargo audit --file src-tauri/Cargo.lock` exits successfully with 18 allowed
  warnings: inherited Tauri/Linux GTK and `unic-*` maintenance advisories, the
  known transitive `glib` warning, and an unmaintained `ttf-parser` warning
  through `pdf-extract`. No vulnerability finding was reported. These remain a
  bounded dependency-refresh follow-up rather than a macOS source-tag blocker.
