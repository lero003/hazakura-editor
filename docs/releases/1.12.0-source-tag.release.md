# Hazakura Editor 1.12.0 Source / Local App Release

> **OKF Starter Scaffold — 知識フォルダの出発点**

Status: Published as a GitHub source / local-app release on
2026-07-17. No binary assets are attached.

`v1.12.0` is the source / local-app checkpoint for the OKF starter
scaffold lane (and the rolled-forward v1.9–v1.11 writing / structure /
OKF review work since the previous public GitHub tag `v1.8.0`).

This release publishes the repository state, source archive, release
notes, and build-from-source path. It does not attach a DMG or App Store
package. The same product line is also **published on the Mac App Store as
`1.12.0`** (user-reported 2026-07-17); see
`docs/releases/1.12.0-app-store-release-notes.md` and
`docs/current-status.md`.

This is not an App Store, App Review-approved, Developer ID signed,
notarized, installer, updater, or production-ready binary distribution.

## Build From Source

```bash
npm ci
npm run build
```

The local smoke bundle is generated at:

```txt
src-tauri/target/release/bundle/macos/Hazakura Editor.app
```

That local app is ad-hoc signed for local validation only. It is not
Developer ID signed or notarized.

## What Changed Since 1.8.0

### v1.12 OKF Starter Scaffold

- Explicit Command Palette, folder context menu, sidebar New menu, and
  File menu actions create a new uniquely named folder with fixed
  **minimal** or **book-like** Markdown templates (OKF v0.1 Draft pin
  `ee67a5c`).
- Materializes the actual local creation date, rejects unclean relative
  paths / NUL content, does not overwrite existing paths, and cleans up
  only artifacts it created on failure.
- Opens `index.md` in the existing Safe Editor tab and invites a separate
  explicit OKF review. No Book Scope, auto-repair, multi-file export, or
  startup scan.

### v1.11 OKF Draft Compatibility Preview

- Explicit, bounded, cancellable, read-only review of one user-selected
  workspace root or subfolder against the same OKF pin.
- Action loop: review findings → open/edit in the existing tab → save →
  recheck. No background indexing or automatic repair.

### v1.10 Single-document Structure Foundation

- Shared source-offset interpretation of closed leading frontmatter, ATX
  headings, standalone page-breaks, and EPUB navigation candidates.
- Outline shows hierarchy and page-breaks, non-blocking structure advice,
  and one-level ATX heading change via a single Undo-able transaction.

### v1.9 Writing Loop Clarity

- Returning Start Panel, pane state language around 書く・読む・確かめる,
  session continuity, and Command Palette ja/kana discoverability.

### Other hardening (selected)

- Preview scroll stability during typing and render scheduling polish.
- Product-boundary and quality documentation updates for agents and
  release gates (including TDD guidance in `AGENTS.md`).

## Release Boundary

v1.12.0 does not add:

- Book Scope, chapter ordering, whole-book export, or multi-file edit
- Startup OKF scan, persistent indexing, or automatic repair
- Agent Workbench / CLI Agent in the App Store lane
- Network fallback, auto-save, tool calling, Git, LSP, terminal, or plugins
- Developer ID signed or notarized binary assets on this GitHub Release

Markdown / text source remains canonical. Safe Editor remains primary.

## Known Limits

- The default local smoke app is ad-hoc signed only.
- Hazakura Local Assist remains on-device, explicit, unsaved, and
  reviewable when Foundation Models is available.
- Image / media remote or outside-workspace loading remains closed by
  default (planned separately; not shipped here).
- Full built-app smoke breadth (IME, long-document, accessibility) is
  manual evidence; see `docs/smoke-checklist.md` and
  `docs/current-status.md`.
