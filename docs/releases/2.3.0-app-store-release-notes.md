## App Store Connect Copy

Hazakura Editor v2.3では、本としての作業を持ち運びやすくし、読書の続きから再開しやすくしました。

・本の章立てを JSON の「章立てレシピ」として書き出せます。別の Mac やバックアップから取り込むときは、必ず下書きとして開き、保存するまで反映しません。
・章立てレシピは相対パスの Markdown 一覧です。OKF 規格ではなく、起動時の自動読込もしません。
・「本全体を読む」では、この Mac のアプリ内に最後に読んだ章とスクロール位置を覚え、次回は続きから開けます。プロジェクトフォルダには書き込みません。
・親フォルダをワークスペースとして開いた場合も、章ファイルからの相対パスで参照した画像がプレビューと「本全体を読む」に安定して表示されるようにしました。
・EPUB書き出し時に、ローカル画像を表紙として明示的に選べます。Markdown内の先頭画像を自動採用せず、選択内容を原稿へ書き戻しません。
・v2.1〜v2.2の本の中検索、章ナビ、書き出し後のFinder表示、Local Assistの状態表示はそのまま引き継ぎます。

Markdown sourceを正本とし、background indexing、auto-save、auto-apply、
縦書きを行わない境界は変わりません。

## Keep Inventory (shipped in `2.3.0`)

| Theme | Keep (in `2.3.0`) |
|---|---|
| Portable recipe | Explicit export/import of relative-path JSON (`hazakura-book-recipe` v1); import is draft-until-Save; never auto-loaded |
| Reader resume | App-private per-workspace chapter + scroll resume for whole-book Reader (max 8 workspaces) |
| Reader search | Bounded whole-book search over loaded chapter names / visible Markdown; `Command+F` / Enter navigation while Reader is open |
| Preview images | Near-viewport deferred load; nested WKWebView fallback; data-URL commit so images do not flash-then-blank |
| EPUB cover | Optional explicit local cover image per export; no first-image inference or source rewrite |
| Recent folders | Per-workspace security-scoped bookmark restore with picker reauth fallback |
| Quality pack | Chapter prev/next, export Finder reveal, Book empty-state honesty, Assist availability honesty, preflight fix hints, e-book find-result sync |

## Boundary

- Recipe import never auto-saves; Save remains explicit.
- Recipe format is `hazakura-book-recipe` v1 with relative Markdown paths only.
- Reader resume is app-private localStorage (up to eight workspaces), not a
  project file and not a second editable buffer.
- Preview image fallback keeps document-relative resolution and the existing
  workspace containment boundary.
- EPUB cover selection is explicit and per export. It does not infer the first
  Markdown image, persist a project setting, or provide a cover editor.
- Vertical writing remains deferred.
- Safe Editor remains primary; Markdown/text source remains canonical per file.
- App Store lane omits Agent Workbench, CLI Agent, arbitrary command execution,
  and external AI/API providers.

## Explicit Non-goals

- vertical writing (縦書き)
- cover cropping / cover editor
- auto-load recipes or workspace manifests
- Compare Center, static lint, mode-pill redesigns
- background indexing, auto-apply, auto-save, second editable buffer

## Product Promise

Markdownで書き、本として読み、ローカルAIで整える — 選んだ章立てを持ち運び、
続きから読み返しやすくした段階です。説明は Help に置き、画面に機能を並べすぎません。
