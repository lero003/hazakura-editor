# Hazakura Editor 1.11.0 TestFlight Candidate Notes

Status: Active TestFlight candidate copy; not published
Scope: App Store Connect copy and release boundary for `1.11.0`
Authority: Medium
Last reviewed: 2026-07-15

`1.11.0` rolls forward the v1.9 writing-flow and v1.10 single-document
structure work, then adds an explicit OKF v0.1 Draft compatibility preview.
The published Mac App Store version remains `1.8.0`. Local package provenance
is recorded in ignored `docs/internal/app-store-candidates/latest.json`;
upload, TestFlight installation, App Review, and publication are separate.

## App Store Connect Copy

Hazakura Editor v1.11は、「書く・読む・確かめる」の流れと、一つのMarkdown文書の構造確認を整え、知識フォルダをOKF v0.1 Draftの観点から明示的に点検できるようにしたリリースです。

・Start Panel、ペイン表示、コマンドパレットを「書く・読む・確かめる」の言葉で整理しました。
・Outlineに見出し階層とページ区切りを表示し、構造上の気づきを修正を強制しない助言として示します。
・見出しレベルは、Undoできる明示的な単一文書の操作として変更できます。保存は引き続き明示操作です。
・コマンドパレットまたはフォルダメニューから「知識フォルダ（OKF）を点検」を実行できます。
・点検結果は「対応が必要」「OKFとして整える準備」「改善案」「参考情報」に分け、該当箇所を既存の編集タブで開いて修正後に再点検できます。
・点検は選択したフォルダに対して明示的に実行され、読み取り量を制限でき、キャンセルできます。起動時の自動走査、バックグラウンド索引、複数ファイルの自動修正は行いません。

Markdown sourceを正本とし、外部AIへのnetwork fallback、auto-save、tool calling、workspace-wide indexingを行わない境界は変わりません。

## Product Boundary

- OKF support is described as a compatibility preview for the pinned v0.1
  Draft snapshot, not full conformance or an OKF-defined book workflow.
- The review uses a bounded disk snapshot. It does not render workspace
  Markdown as HTML or load its images inside the result panel.
- `開いて修正` uses the existing Safe Editor tab. The review does not apply
  automatic repairs or write multiple files.
- Book Scope, chapter ordering, whole-book export, persistent indexing, and
  background scanning remain out of scope.
- The App Store lane omits Agent Workbench, CLI Agent, arbitrary command
  execution, and external AI/API providers.
