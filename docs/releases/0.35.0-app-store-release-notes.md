Status: Draft
Scope: User-facing App Store / TestFlight release-note copy for `0.35.0`
Authority: Medium
Last reviewed: 2026-06-26

This note is the public-facing release-note draft for the `0.35.0` App
Store / TestFlight recovery candidate. It does not by itself claim App
Store Connect upload, Apple processing, TestFlight distribution, App
Review approval, or public Mac App Store release.

## App Store Connect Copy

PDF書き出しと、長文表示・編集の安定性を改善しました。

- PDF出力をmacOSの印刷ダイアログ経由ではなく、アプリ内の直接書き出しに変更しました。
- TestFlight環境でも外部ブラウザや印刷UIに依存しないよう、保存先を選んで `.pdf` を生成する方式にしました。
- 出力されるPDFを1枚の長いページではなく、A4サイズのページに分割するようにしました。
- プレビューや電子書籍モードの長文表示が重くなりにくいよう、描画とページ計測を調整しました。
- プレビューの慣性スクロールや、エディタのスクロールバー操作後の挙動を改善しました。
- Local Assistの提案を適用するとき、対象タブをより確実に扱うようにしました。
- ワークスペース画像が電子書籍モードで透明のまま表示されることがある問題を修正しました。
- EPUB書き出し、ワークスペースの開いているファイル表示、右クリックからのMarkdown挿入など、前回までの改善も含まれています。

## Longer Release Note

`0.35.0` supersedes the `0.34.0` native print candidate. The local
native-print smoke passed, but the TestFlight build still showed macOS'
"This application does not support printing" alert. This release
therefore removes the user-facing dependency on the macOS print UI for
PDF output.

Main changes:

- PDF output now uses a save dialog plus direct app-owned PDF generation.
- The app renders the generated HTML in an app-owned WebView and asks
  WebKit to create PDF data, then writes that data to the selected
  `.pdf` file.
- Exported PDF content is paginated into A4-sized pages instead of one
  long single page, so the result opens and prints naturally.
- The user-facing path no longer opens the OS default browser, a shell,
  an external opener, or the macOS print dialog.
- Rust-side validation requires the main window, non-empty HTML, an
  existing parent folder, and a `.pdf` destination.
- Preview scroll synchronization avoids fighting inertial scrolling.
- E-book rendering debounces repeated Markdown sanitization after the
  initial render and coalesces page measurement into animation frames.
- Markdown rendering performs fewer HTML parse passes.
- Editor scrollbar and Go to Line focus behavior was hardened.
- Local Assist applies generated text to the intended tab more
  explicitly.
- Workspace images that could stay transparent in e-book Mode are
  remeasured more reliably.

## Boundary

This release does not add a generic browser launcher, shell input,
Preview DOM editing, automatic rewriting, auto-save, Git integration,
LSP, terminal, arbitrary command execution, external AI/API calls, or
Agent Workbench to the App Store lane.

PDF export remains an explicit user action. Hazakura Local Assist
remains only an explicit, on-device, availability-gated writing
companion whose result stays unsaved until accepted and reviewable
through Diff / Discard.

## Verification

Source proof should include focused PDF export hook tests, Rust
main-window / `.pdf` destination guard tests, full frontend and Rust
test suites, App Store surface smoke, signed distribution probe,
package signature check, sandbox preview smoke, Info.plist
version/build checks, package SHA check, and `git diff --check`.

TestFlight install / launch and confirming that PDF export creates the
selected file remain required before public release.
