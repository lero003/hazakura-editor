# Hazakura Editor 0.29.1 App Store Submission / Release Evidence

> **Helper-enabled App Store lane package and release note**

Status: local package evidence generated on 2026-06-22; App Review
approval and public Mac App Store release reported by the user on
2026-06-23. Raw App Store Connect, TestFlight, and App Review logs are
handled outside this repository unless separately recorded.

`0.29.1` packages the v0.29.01 Hazakura Local Assist responsiveness lane
after the earlier `0.29.0` helper-enabled package candidates. Build `33`
supersedes build `31` as the latest local package evidence after the
Markdown preview flicker fix.

The release boundary stays unchanged: Markdown source remains canonical,
Safe Editor remains primary, and the App Store lane omits Agent
Workbench, CLI Agent launch, dev mode, arbitrary command execution,
external AI/API calls, Git, LSP, terminal, plugins, workspace-wide
indexing, auto-apply, and auto-commit behavior. Hazakura Local Assist is
included only as an explicit, on-device writing companion whose final
result enters the existing unsaved edit transaction / Diff review flow.

## v0.29.1 Release Focus

- Hazakura Local Assist generation is request-scoped, emits progress before
  the heavy local model wait, and ignores stale request events.
- The target editor is read-only while its Local Assist request is
  generating, while the app shell and Assist Window remain usable.
- Streaming partial output appears only in the Assist Window preview; the
  editor buffer changes only after the final candidate enters Diff review.
- The helper protocol supports `candidate_partial` envelopes and keeps the
  final `candidate` envelope for compatibility.
- Local Assist prompts are shorter and preset wording is simplified for the
  current on-device model quality.
- Assist settings now make Local Assist availability more visible while
  preserving the preview / on-device / no external AI service boundary.
- Markdown preview rendering now keeps the existing preview visible while
  same-document edits wait for the next render frame, reducing visible
  flicker during split edit / preview work.
