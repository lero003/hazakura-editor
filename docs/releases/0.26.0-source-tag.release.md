# Hazakura Editor 0.26.0 Source / Local App Tag

> **no-workspace authoring and EPUB export beta**

Status: Prepared for a GitHub source / local-app tag on 2026-06-20. No
binary assets are attached by this note.

`v0.26.0` is the source / local-app checkpoint for no-workspace New File
/ Save As, e-book affordance polish, and the first practical EPUB export
beta. Markdown source remains canonical, and EPUB export is an explicit
active-document action rather than a second saved document model.

This release publishes the repository state, source archive, release
notes, and build-from-source path. It does not attach a DMG asset. The
latest downloadable warning-expected DMG preview remains `v0.20.0`.

The App Store lane remains helper-free and omits Agent Workbench, CLI
Agent launch, Apple Local Assist helper, external AI/API calls, and
arbitrary command execution surfaces.

This is not a Developer ID signed, notarized, updater, or installer-grade
distribution.

## What Changed Since 0.25.0

### No-workspace Authoring

- New File works without a selected workspace by opening a pathless
  untitled editor tab.
- Save on a pathless tab routes through Save As, then rekeys the tab to
  the selected file path.
- Pathless tabs are not restored as workspace files and are not written
  into hidden workspace draft state.

### e-book Mode Polish

- The e-book toggle stays visible but disabled/inactive when no document
  is available or an image preview is foregrounded.
- Chapter-boundary page movement no longer animates as a backwards slide
  when moving to the next chapter.
- Explicit page-break hints from blank-line-flanked standalone `---` or
  `===` lines are shown in e-book Mode while preserving Markdown source.

### EPUB Export Beta

- Adds `Export EPUB (Beta)...` from the File menu and command palette.
- Exports the active Markdown document to a minimal EPUB 3 archive with
  XHTML content, generated heading navigation, a small stylesheet, and
  dialog-scoped Title / Author / Language metadata.
- Packages readable workspace images and allowed small `data:image`
  references into EPUB resources, while replacing unavailable images with
  an in-document warning.
- Strips Preview-only markup before XHTML output, ignores YAML
  frontmatter for export navigation/content, generates valid per-export
  UUID identifiers, and writes `dcterms:modified` from export time.
- Treats blank-line-flanked standalone `---` / `===` lines as explicit
  page-break hints in the EPUB output.
- Allows explicit overwrite after the native Save As dialog confirms the
  chosen export path.
- Adds gentle table borders to the EPUB stylesheet.

## Release Boundary

v0.26.0 does not add:

- A new GitHub DMG asset.
- Developer ID notarization, stapling, auto-update, or installer-grade
  distribution.
- Cover asset management, vertical writing, multi-file book ordering,
  reader-perfect pagination, or in-app validator launching.
- Preview-DOM editing or a separate EPUB document model.
- New Agent Workbench capability, provider plugins, command execution,
  terminal, LSP, Git integration, workspace-wide indexing, auto-apply, or
  auto-save.

## Known Limitations

- EPUB export is beta and reader pagination remains reader-dependent.
- EPUBCheck is a manual outside-app step; the app does not launch
  external validators.
- Metadata entered in the EPUB dialog is per-export only and is not
  written back to Markdown source or persistent document state.
- Cover selection, vertical writing, multi-file books, and advanced EPUB
  navigation are deferred to later planning.
- The latest downloadable warning-expected DMG preview remains
  `v0.20.0`; users who need a binary preview should use that release
  until a later DMG preview is published.
