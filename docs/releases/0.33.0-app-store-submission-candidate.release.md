# Hazakura Editor 0.33.0 App Store / TestFlight Candidate

Status: local package evidence generated on 2026-06-25 for TestFlight-side
v0.33 EPUB Export v1 Polish smoke. App Store Connect upload, processing,

performed outside this repository unless separately recorded.

`0.33.0` packages the v0.33 EPUB Export v1 Polish plus the v1 workspace
marker and right-click slash-command fit-and-finish, and the v1.0 RC
pre-RC quality-slice planning pass. The release boundary stays unchanged:
Markdown source remains canonical, Safe Editor remains primary, and the
App Store lane omits Agent Workbench, CLI Agent launch, dev mode,
arbitrary command execution, external AI/API calls, Git, LSP, terminal,
plugins, workspace-wide indexing, auto-apply, and auto-commit behavior.
Hazakura Local Assist remains only an explicit, on-device writing
companion whose final result enters the existing unsaved edit
transaction / Diff review flow.

## v0.33 Candidate Focus

- EPUB export is presented as `EPUB書き出し` / `EPUB Export` instead of
  beta copy. It remains an explicit active-document action over Markdown
  source.
- Successful exports that replaced some images now report a warning
  status rather than a silent success (`image-unavailable`).
- Generated EPUB nav/content XHTML uses the selected language metadata
  instead of hardcoding `ja`.
- The workspace tree shows existing-tab-derived open and dirty markers
  for files inside the selected workspace.
- The editor content area opens the existing slash-command menu from a
  right-click, preserving selection when invoked inside the selection.
