# hazakura editor 0.13.0 Source / Local App Tag

> **Distribution Probe / L Mode Bridge**

v0.13.0 closes the first App Store / Developer-GitHub distribution-probe pass and records the current L Mode peer-mode polish as a source / local-app checkpoint.

This tag is a source / local-app checkpoint. It is not a signed, notarized, App Store, TestFlight, or warning-expected DMG publication.

The latest published downloadable DMG preview remains `v0.11.0`.

## What Changed Since 0.12.0

### Distribution Probe

- Makes `npm run build` produce the App Store preview / normal lane.
- Adds `npm run build:macos-lanes` to also produce a separate Developer / GitHub lane bundle named `hazakura editor Dev.app`.
- Re-identifies the Developer / GitHub lane as bundle identifier `lab.hazakura.note.dev` and marks it with a small `DEV` badge.
- Keeps the warning-expected DMG preview lane pointed at the Dev bundle.
- Hides or omits Agent Workbench settings, menu entries, Command Palette actions, and `agent.html` from the App Store preview lane.
- Rejects Agent Workbench IPC on the Rust command surface when compiled for the App Store distribution lane.
- Adds draft App Store sandbox entitlements for the app and Hazakura Local Assist helper.
- Proves that an ad-hoc sandbox-signed Tauri parent can spawn the inherited-sandbox Hazakura Local Assist helper and receive an availability envelope.

### L Mode Bridge

- Keeps the file tree hidden by default in L Mode, with a temporary workspace drawer available from a translucent top-left control.
- Opens dirty-buffer review as a local floating diff sheet in L Mode.
- Labels the L Mode escape route as `編集モード` rather than implying termination.
- Adds an explicit `えるモード` route from normal edit mode.
- Groups normal-mode `変更を確認` / `差分` / `アウトライン` under the compact `確認` menu while keeping `プレビュー` visible.
- Exposes Typewriter mode from L Mode without making it a hidden preference trap.
- Tightens dark / yakou diff contrast.
- Keeps active list bullets / ordered numbers visible while the cursor is on a list line.
- Renders Setext-style `---` / `===` underline markers as the same divider widget so typing on the line before a divider does not make the visual boundary disappear.

### Planning Boundary

- Moves the deeper L Mode WYSIWYG Accuracy Ramp to v0.14.
- Records the v0.14 direction in `docs/l-mode-plan.md`, `docs/roadmap.md`, and `docs/current-status.md`.
- Keeps the saved Markdown source as the canonical document model.

## Release Boundary

v0.13.0 does not add:

- App Store or TestFlight submission.
- Apple Developer / App Store signing, provisioning, upload validation, or review approval.
- Developer ID signing, notarization, installer packaging, or updater.
- A GitHub Release or DMG asset for this tag.
- A completed WYSIWYG editing model.
- Preview DOM editing, `contenteditable`, HTML as the saved canonical model, or hidden save-time rewriting.
- Network-backed LLM fallback.
- New Agent Workbench capability, provider plugins, command execution, terminal, LSP, Git integration, or workspace-wide indexing.

## Known Limitations

- The App Store preview lane is a structural preview only. It is not App Store-ready until Apple Developer / App Store signing, provisioning, upload validation, and review behavior are proven.
- Dependency/package-level pruning and capability-profile separation for the App Store lane remain future hardening.
- The sandbox proof is ad-hoc and local. It does not replace App Store signing or upload validation.
- Direct shell execution of the inherited-sandbox Hazakura Local Assist helper still fails with the expected inherited-sandbox shape; parent-spawn behavior is the meaningful proof for the current sidecar design.
- L Mode WYSIWYG fidelity still needs a larger v0.14 pass for IME, caret movement, hidden markers, lists, links, tables, images, selection/copy behavior, and visual overlap.
- Hazakura Local Assist live generation remains alpha / experimental and depends on macOS 26+ Apple Foundation Models availability, local Apple Intelligence state, and supported language / locale.
- The local `.app` produced by `npm run build` is ad-hoc signed only and is not Developer ID signed or notarized.

## Local Verification Before Tagging

Required gates for this source / local-app tag:

```bash
npm ci
npm run typecheck
npm run test
npm run build:vite
cargo fmt --manifest-path src-tauri/Cargo.toml -- --check
cargo test --manifest-path src-tauri/Cargo.toml
npm run build
npm run build:macos-lanes
npm run smoke:macos-sandbox-preview
git diff --check
npm audit
cargo audit --file src-tauri/Cargo.lock
```

The built app must report version `0.13.0`, bundle identifier `lab.hazakura.note`, product name `hazakura editor`, and executable `hazakura-editor`.

The built app must pass:

```bash
codesign --verify --deep --strict --verbose=2 "src-tauri/target/release/bundle/macos/hazakura editor.app"
```

Expected warning for the local ad-hoc app:

```bash
spctl -a -vv -t open "src-tauri/target/release/bundle/macos/hazakura editor.app"
```

Gatekeeper may reject the app with insufficient-context / not-notarized style output because this tag does not produce a notarized distribution asset.
