## App Store Connect Copy

Hazakura Editor v1.13は、操作できない理由や読み込み中の状態を分かりやすくし、画像を安全にプレビュー・書き出すための選択肢を整えたリリースです。

・コマンドパレットで、現在使えない操作とその理由を確認できるようにしました。
・Quick Openの検索範囲と件数上限、検索結果を開いた状態、遅いPreview・電子書籍読み込みを明示します。
・Quick Settingsのキーボード操作とフォーカス復帰、OKFひな形の段階表示、ゴミ箱へ移す対象の案内を整えました。
・設定画面を広いウインドウでは見渡しやすく、狭いウインドウでは一列のまま読める配置にしました。
・workspace外のローカル画像は文書ごとの明示許可、リモート画像は既定オフのHTTPS専用設定として扱います。
・PDF/EPUB書き出し時は、許可された画像を出力へ実体化しやすくしました。Markdown sourceは自動で書き換えません。
・画像をworkspaceの`assets/`へ明示的に固定する操作は、Undoできる単一の編集として扱います。

Markdown sourceを正本とし、外部AIへのnetwork fallback、auto-save、tool calling、workspace-wide indexingを行わない境界は変わりません。

## Release Position

`1.13.0` is the first post-v1.12 refinement release. It carries the kept
Signal & Silence interaction slices and the Theme G media-boundary work,
including explicit consent, HTTPS-only opt-in remote images, export
materialization, and an Undo-able pin-to-assets action. It does not add Book
Scope, background indexing, automatic source rewriting, or a second editable
document model.

The product promise is unchanged:

> Markdownで書き、本として読み、ローカルAIで整える。

## Boundary

- Safe Editor remains primary; Markdown/text source remains canonical.
- Outside-workspace and remote image access remains explicit and bounded.
- Export materialization does not silently rewrite the source document.
- App Store lane omits Agent Workbench, CLI Agent, arbitrary command
  execution, and external AI/API providers.
- Hazakura Local Assist remains on-device, explicit, unsaved, and reviewable.

## Explicit Non-goals

- Book Scope, chapter ordering, or whole-book state
- Background indexing or startup scans
- Automatic source rewrite or automatic remote-image loading
- Git, LSP, terminal, plugins, or arbitrary command execution
