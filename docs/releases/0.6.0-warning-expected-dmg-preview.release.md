# hazakura editor 0.6.0 Warning-expected DMG Preview

> **Foundation Release / Daily-Drivable Safe Editor**

v0.6.0 is a foundation release for making the safe editor practical for daily use and for renaming `hazakura-note` to `hazakura editor`.
It closes the v0.6 refactor boundary, keeps Safe Editor primary, and keeps Agent Workbench optional and explicitly gated.

This is a warning-expected macOS DMG preview. The app is ad-hoc signed and not notarized, so Gatekeeper warnings are expected.

Only use this preview if you are comfortable evaluating unsigned and not-notarized macOS preview binaries.

## What Changed Since 0.5.0

### Safe Editor Foundation

- Renamed the app-facing product identity to `hazakura editor`; release artifacts use the `hazakura-editor` basename.
- Continued the App.tsx split into focused hooks and shell components without introducing Context or zustand.
- Added Cmd+P Quick Open for workspace files.
- Added workspace auto-backups for dirty tabs, with an ON/OFF toggle in Preferences > Application.
- Fixed Replace one / Replace all to use current search matches and allow empty replacement text.
- Added pointer-based tab reordering for multiple open files.
- Added multi-cursor and rectangular selection support through CodeMirror behavior.

### Agent Workbench Boundary

- Kept Agent Workbench optional, gated, and provider allowlisted.
- Improved Agent output polling so terminal output updates without repeatedly appending old chunks.
- Added selected-text send and preset prompt chips while preserving the Safe Editor boundary.

### Stability And Safety

- Hardened auto-backup paths so backups stay under the selected workspace backup directory.
- Preserved image asset preview/export behavior from the v0.5 follow-up work.
- Continued localized Japanese/English settings and status copy polish where it supports release-facing flows.

## Known Limitations

- This release does not add Git integration, LSP, project-wide indexing, arbitrary command execution, auto-apply, or auto-commit behavior.
- Agent Workbench provider behavior remains outside hazakura's safety boundary.
- Alt text editing for inserted images remains a future authoring improvement.
- The JavaScript bundle remains large; code splitting is planned for a future release.
- Tag publication and remote asset verification are still pending.

## Local Verification Before Publication

Completed locally before publication:

```bash
npm run typecheck
cargo test --manifest-path src-tauri/Cargo.toml
npm run build
npm run build:dmg-preview
cd src-tauri/target/release/bundle/dmg
shasum -c hazakura-editor_0.6.0_aarch64-warning-expected.dmg.sha256
```

The built and mounted apps reported version `0.6.0`, bundle identifier `lab.hazakura.note`, product name `hazakura editor`, and executable `hazakura-editor`.

Focused built-app smoke also confirmed launch, temp-file open/save, Export as HTML, split Diff comparison, tab drag reorder, previous/next tab focus, Replace all, Cmd+P Quick Open, Preferences > Application auto-backup ON/OFF, and a real auto-backup write under `.hazakura/backups/`.

The built app passed:

```bash
codesign --verify --deep --strict --verbose=2 "src-tauri/target/release/bundle/macos/hazakura editor.app"
```

Expected warning confirmed:

```bash
spctl -a -vv -t open "src-tauri/target/release/bundle/macos/hazakura editor.app"
```

The app was rejected by Gatekeeper with `source=Insufficient Context`, which is expected because this preview is ad-hoc signed and not notarized.
