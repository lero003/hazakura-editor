## Release Position

`1.3.0` is the v1.3 Daily Trust release. It does not add a new product
surface, a second document model, arbitrary command execution, an external
AI/API path, or a broader workspace model. It deepens the existing
single-document Safe Editor with daily-use trust through four independent
slices.

The product promise is unchanged:

> Markdownで書き、本として読み、ローカルAIで整える。

The core flow remains write -> read -> notice -> return -> revise ->
review -> export. Markdown source remains canonical throughout that flow.

## What Changed In v1.3

- **Save As session continuity.** Save As keeps the same open-tab /
  CodeMirror session for the same language family, so selection, scroll,
  undo, and redo survive a path change. A real Markdown-to-HTML/CSS
  language-family change may still remount CodeMirror so the parser stays
  correct; the tab id stays stable while the document path and view-state
  key migrate.
- **Explicit Local Assist review.** Hazakura Local Assist review now uses
  explicit `採用` / `破棄`. `採用` applies only to the reversible Local
  Assist transaction and keeps the accepted buffer dirty and unsaved; it
  never saves automatically. File Diff stays read-only and Recovery
  retains its explicit restore action.
- **Reading Focus TOC context.** The Reading Focus table-of-contents
  drawer shows up to two H3+ subheadings, a remaining count, and measured
  page progress for the current chapter only. The H3+ text is context,
  not a new approximate page-jump target.
- **A4 PDF margin presets.** PDF export opens an A4 margin chooser with
  `狭い` (10/10 mm), `標準` (18/16 mm), and `広い` (25/22 mm). The preset
  is request-scoped and no arbitrary CSS, numeric margin field, page-size
  / DTP control, or macOS print UI is exposed. PDF stays A4-only; the
  existing Rust main-window, non-empty HTML, parent-directory, and `.pdf`
  destination guards remain unchanged.

## Boundary

Hazakura Local Assist remains a preview, availability-gated on-device
writing companion. It has no external AI/API fallback, background
rewriting, auto-save, tool calling, or workspace-wide indexing. Its
proposals remain explicit, unsaved, and Diff / Discard reviewable.

EPUB export remains an explicit single-document workflow. `1.3.0` does
not claim advanced metadata, cover design, navigation editing, multi-file
book management, or a full EPUB production environment. e-book Mode is a
reading preview, not a guarantee that every EPUB reader will reproduce
the same layout.

The App Store lane excludes Agent Workbench, CLI Agent launch, arbitrary
command execution, external AI/API calls, and provider-add UI. The A4
PDF margin presets do not add arbitrary CSS, a numeric margin field, a
page-size control, or a macOS print path; they are an allowlist over the
existing direct PDF export.

- Focused tests cover Save As session continuity, explicit Local Assist
  `採用` / `破棄`, Reading Focus TOC context, and the A4 PDF margin

- The helper-enabled built app passed representative checks: Save As

  As reversed the pre-save sentinel and kept the new path dirty); Local

  unsaved; Reading Focus TOC showed H3/H4 context and current measured

  insets for all three A4 presets, and a standard-margin export went from
  three content plus three blank pages to three content pages.

Built-app extended RC interaction breadth (non-empty selection / scroll /
