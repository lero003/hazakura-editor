## Release Position

`1.10.0` is the v1.10 Single-document Structure Foundation TestFlight
candidate. It builds one source-offset interpretation of closed leading
frontmatter, ATX headings, standalone page-breaks, and EPUB navigation
candidates; the existing Outline now shows heading hierarchy and
page-breaks, exposes non-blocking structure suggestions, and allows only
an explicit one-level ATX heading change through one Undo-able CodeMirror
transaction. It keeps Safe Editor invariants: no cloud OCR, no auto-save,
no second document model, and no Git / LSP / terminal / plugin expansion.

It also carries the v1.9 Writing Loop Clarity work, which organizes
Preview / Reference / e-book / Outline / Diff / L Mode around
`書く・読む・確かめる` without adding chrome or a new model.

The product promise is unchanged:

> Markdownで書き、本として読み、ローカルAIで整える。

## What Changed

### User-visible improvements (v1.10 Single-document Structure Foundation)

- **Shared structure model.** `parseMarkdownStructure` provides one
  source-offset interpretation of closed leading frontmatter, ATX
  headings, standalone page-breaks, and EPUB navigation candidates.
  Outline, e-book chapter splitting, and EPUB export share this model.
- **Structure overview.** The existing Outline surface shows heading
  hierarchy and page-breaks as read-only rows. Selecting a row moves the
  Editor to its source line without changing the source or dirty state.
  No new pane or chrome was added.
- **Advisory checks.** Skipped heading level, empty heading, duplicate
  navigation label, and an 800-line / 40,000-character section are shown
  as non-blocking advice. These are notes only; Save, Save As, Preview,
  e-book, PDF, and EPUB actions remain available.
- **One bounded edit.** A heading level change is the only demonstrated
  structure edit: one explicit, Undo-able single-buffer CodeMirror
  transaction. The tab becomes dirty and no file write occurs. One
  `Cmd+Z` restores the original source and clean state.

### User-visible improvements (v1.9 Writing Loop Clarity)

- **Returning Start Panel.** First-use and returning headers are
  separated; previous-workspace resume and pathless draft recovery are
  gathered on the Start Panel. Resume tries path, then bookmark, then
  picker.
- **Pane state copy.** Preview / e-book / Outline / Diff / Reference
  show/hide/retained state is explained through toggle titles and status
  in `読む・確かめる` wording; empty-state copy follows the same voice.
- **View state continuity.** Editor / e-book / Preview position state is
  held by `sessionId` rather than path, removing the Save As rekey race.
  Tab-switch isolation is pinned by existing tests.
- **Command palette discoverability.** The palette is organized into
  `書く・読む・確かめる`, with label and keywords aligned across
  English / Japanese / kana.

### Trust / maintenance

- No manifest, second buffer, background scan, automatic correction,
  section move, or Book Scope was added. Structure interpretation stays
  inside one Markdown source.
- CodeMirror `@codemirror/view` remains pinned at **6.43.2** for display
  stability on long Japanese Markdown.
- The structure edit stays a single-buffer transaction with Undo, dirty
  state, CRLF metadata, stale-offset, and IME regression coverage.

## Boundary

- Safe Editor remains primary; Markdown/text source remains canonical.
- App Store lane omits Agent Workbench, CLI Agent, arbitrary command
  execution, and external AI/API providers.
- Hazakura Local Assist remains on-device, explicit, unsaved, and
  reviewable.
- One editable Markdown document may be shown beside one read-only
  reference; this is not a generic split editor or second document model.
- Structure editing stays limited to one explicit Undo-able heading-level
  change in the active buffer. Multi-file Book Scope remains v2.

## Follow-up Verification

`1.10.0` is a TestFlight candidate, not a published release. The
following remain useful extended verification rather than publication
blockers:

- Source-jump breadth, IME composition, Save As, recovery, and e-book /
  EPUB structure-alignment smoke from `docs/smoke-checklist.md`.
- Signed TestFlight interaction breadth and spoken VoiceOver across the
  structure overview, advisory, and heading-level edit.
- The OKF Draft Compatibility Preview and larger cross-surface
  distribution-confidence matrix planned for v1.11 before v2 Book Scope.

## Explicit Non-goals

- Cloud OCR or external document upload
- Auto-save or auto-apply of import or assist results
- Two editable panes, Review Desk revival, or multi-file Book Scope
- Background workspace scan, hidden manifest, or second document model
- Automatic structure correction, section move, or heading rewrite
- Git, LSP, terminal, plugins, or arbitrary command execution
