## Release Position

`1.5.0` is the v1.5 release. It is a stabilization and reading-polish arc:
four small slices (Settings surface consistency, dead-code sweep, dependency
updates, Reading Focus TOC density), one visual polish slice (CRT noise
intensification), one window chrome fix (traffic light position alignment),
and the v1.5 / v1.6 / v2 roadmap documentation. It does not add a new
product surface, a second document model, arbitrary command execution, an
external AI/API path, or a broader workspace model.

The product promise is unchanged:

> Markdownで書き、本として読み、ローカルAIで整える。

The core flow remains write -> read -> notice -> return -> revise ->
review -> export. Markdown source remains canonical throughout that flow.

## What Changed In v1.5

### User-visible improvements

- **Spellcheck toggle in Preferences.** The Spellcheck toggle that previously
  lived only in the editor Quick Settings popover is now also available in the
  main Settings preferences pane, so display / editing preferences are in one
  place. It reuses the existing `spellcheckEnabled` setting; no new editor
  behavior is added.
- **Reading Focus TOC density.** The Reading Focus table-of-contents drawer
  shows denser chapter context: subheading preview expanded from 2 to 4
  entries, page progress on all chapters (not just the current one) when the
  page count is known, a wider and taller panel, and the subheading area now
  allows up to 2 lines instead of a single truncated line. The TOC no longer
  reads as a bare chapter-number list.
- **CRT noise intensification.** The CRT joke theme's procedural noise
  (snow / grain), glitch bands, scanlines, and flicker are now stronger and
  more visible, especially on low-contrast displays. Intensity mapping was
  re-tuned (normal 1.1 -> 1.2, dramatic 1.5 -> 1.8) and noise coefficients
  are now non-linear (`pow(u_intensity, 1.6)`) so noise contrast rises
  relative to background brightness. A new horizontal noise-line burst and a
  whole-frame flicker were added. Shader magic numbers were parameterized as
  named constants for easier tuning.
- **Traffic light position alignment.** The macOS traffic light (close /
  minimize / zoom) position is now `x:9, y:18`, and the `.tabs-row` theme
  transition was removed to eliminate the perceived shift that occurred on
  theme switch and tab operations. This was a Tauri / WKWebView overlay
  timing issue; aligning the configured position with the post-reflow
  position removes the visible drift.

### Maintenance slices

- **Dead code sweep.** Removed unused exports (`TableIcon`,
  `pinRecentEntry` / `unpinRecentEntry`, `APPLE_ASSIST_ALL_OPERATIONS`,
  `APPLE_ASSIST_MAX_INSTRUCTION_CHARS`, `isAppleAssistAvailable`) and unused
  CSS classes (`.start-recent*`, `.find-row`). About 150 lines removed with
  zero additions; no behavior change.
- **Dependency updates.** Brought 37 npm packages up to their latest
  patch / minor versions. DOMPurify and marked were already current. No
  major bumps; `npm outdated` is empty and `npm audit` reports 0
  vulnerabilities.

### Documentation

- **Roadmap clarification.** `docs/current-work.md` and `docs/roadmap.md`
  now explicitly sequence the post-v1.4 lanes: v1.5 (this release),
  v1.6 (Import Assist Phase 1 / Vision OCR), v2 (OKF-based structural
  foundation for Book Scope, then vertical writing). Vertical writing moved
  from v1.x candidate to v2.

The Markdown source contract, export output, and App Store lane boundary are
unchanged. v1.5 adds no new command surface, no DOM editing in Preview, no
second render path, and no telemetry or network reporting.

## Boundary

Hazakura Local Assist remains a preview, availability-gated on-device
writing companion. It has no external AI/API fallback, background
rewriting, auto-save, tool calling, or workspace-wide indexing. Its
proposals remain explicit, unsaved, and Diff / Discard reviewable.

EPUB export remains an explicit single-document workflow. `1.5.0` does not
claim advanced metadata, cover design, navigation editing, multi-file book
management, or a full EPUB production environment. e-book Mode is a reading
preview, not a guarantee that every EPUB reader will reproduce the same
layout.

The App Store lane excludes Agent Workbench, CLI Agent launch, arbitrary
command execution, external AI/API calls, and provider-add UI. The v1.5
slices add no new command surface, no DOM editing in Preview, no second
render path, and no telemetry or network reporting.

The CRT noise intensification is a presentation-only change to an existing
joke theme. It does not add a network path, a new file format, a new export
target, or any telemetry.

`1.5.0`. The signed `1.5.0` App Store / TestFlight package has not been
built yet.

- `npm run test`, `cargo test`, `npm run build:vite`, `npm run build`,
  `cargo fmt --check`, and `git diff --check` pass.
- `npm outdated` is empty; `npm audit` reports 0 vulnerabilities.

  focused test that toggles `spellcheckEnabled` from the preferences pane.
- Reading Focus TOC density coverage: `EBookPane.test.tsx` covers the
  4-entry subheading preview and the current-chapter page progress; the
  non-current-chapter page progress stays null until the chapter is measured.
- CRT noise coverage: `CrtShaderOverlay.test.tsx` (unchanged; the shader
  internals are WebGL-dependent and verified by the existing off / on /
  reduced-motion / `off -> on` revival tests).

  `trafficLightPosition: { x: 9, y: 18 }`.

Uploading the `1.5.0` candidate to App Store Connect is a separate
explicitly gated step. Post-release issues enter the observation queue only
when they are reproducible or indicate a hotfix risk.
