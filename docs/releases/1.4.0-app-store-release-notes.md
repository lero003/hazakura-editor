## Release Position

`1.4.0` is the v1.4 Observability And Testability release. It does not add a
new product surface, a second document model, arbitrary command execution,
an external AI/API path, or a broader workspace model. It deepens the
existing single-document Safe Editor by lifting previously module-private
pure logic into named, focused-test seams.

The product promise is unchanged:

> Markdownで書き、本として読み、ローカルAIで整える。

The core flow remains write -> read -> notice -> return -> revise ->
review -> export. Markdown source remains canonical throughout that flow.

## What Changed In v1.4

- **E-book reader navigation seams.** The page-commit decision (same-chapter
  floor vs. chapter-change no-floor) and the chapter-cross page-target
  computation were pure functions living privately inside `EBookPane.tsx`.
  They are now exported from `ebookPageTarget.ts` and covered by focused
  tests, so the floor and continuation logic no longer depends on DOM-level
  component tests alone.
- **E-book reader-location seams.** The reader-location assembly,
  location equality, page-to-source-line interpolation, and markdown
  line-counting helpers moved to `ebookReaderLocation.ts` with focused
  coverage for trailing-newline / CRLF / single-line edge cases.
- **PDF export layout tests.** The A4 screen-page-layout calculation (margin
  preset -> content box / column gap in PostScript points) and the point
  value formatter gained fixed-value coverage alongside the existing margin
  and table-annotation tests.
- **EPUB export text-helper seams.** The slugify, YAML-frontmatter stripper,
  XML escaper, document-name title fallback, and image media-type mappers
  moved to `epubTextHelpers.ts` with focused coverage for Japanese-heading
  and frontmatter edge cases.

No user-visible behavior, Markdown source contract, export output, or App
Store lane boundary changed. Every slice is a pure-logic extraction with new
focused tests; the existing component / integration tests serve as the
regression safety net.

## Boundary

Hazakura Local Assist remains a preview, availability-gated on-device
writing companion. It has no external AI/API fallback, background
rewriting, auto-save, tool calling, or workspace-wide indexing. Its
proposals remain explicit, unsaved, and Diff / Discard reviewable.

EPUB export remains an explicit single-document workflow. `1.4.0` does not
claim advanced metadata, cover design, navigation editing, multi-file book
management, or a full EPUB production environment. e-book Mode is a reading
preview, not a guarantee that every EPUB reader will reproduce the same
layout.

The App Store lane excludes Agent Workbench, CLI Agent launch, arbitrary
command execution, external AI/API calls, and provider-add UI. The v1.4
observability slices add no new command surface, no DOM editing in Preview,
no second render path, and no telemetry or network reporting.

## Verification Boundary

Local source, build, and audit gates form the candidate evidence for
`1.4.0`. The signed `1.4.0` App Store / TestFlight package has not been
built yet.

- `npm run test`, `cargo test`, `npm run build:vite`, `npm run build`,
  `cargo fmt --check`, and `git diff --check` pass.
- Focused tests cover the four extracted seams (page target, reader
  location, PDF layout, EPUB text helpers). The existing `EBookPane.test.tsx`
  and `epubExport.test.ts` suites pass unchanged, confirming no
  behavior regression.

Building, signing, and uploading the `1.4.0` candidate is a separate
explicitly gated step. Post-release issues enter the observation queue only
when they are reproducible or indicate a hotfix risk.
