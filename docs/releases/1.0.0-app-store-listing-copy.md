## Description

Hazakura Editorは、Markdownで文章を書き、電子書籍のように読み返し、必要な部分だけをローカルAIと整えられるmacOS向け執筆エディタです。

メモ帳より少し頼れて、IDEほど大きくない。文章のsourceを自分で持ちながら、書く、読む、戻って直す、提案を比べる、書き出す流れを、ひとつの静かな作業場にまとめます。

● Markdownで、静かに書く

通常のMarkdown編集に加えて、文章の流れに集中しやすいL Modeを用意しています。sourceは常にMarkdownのまま。表示のために本文を別形式へ書き換えません。

● 電子書籍のように読み返す

Markdownプレビューに加えて、ページ送りや見開き表示を備えた電子書籍モードで長い文章を読み返せます。気づいた場所から対応するMarkdownへ戻り、同じ原稿を直し続けられます。

● ローカルAIの提案を、差分で確認する

Hazakura Local Assist（プレビュー）は、利用可能なMac上のオンデバイスモデルを使う文章支援です。校正、要約、翻訳、短縮、続きを考えるための案などを依頼できます。

提案は自動保存されません。変更前後をDiffで確認し、採用するか破棄するかを自分で決められます。外部AIサービスへのnetwork fallback、background rewriting、tool calling、workspace-wide indexingは行いません。

Local Assistの利用可否は、Mac、macOS、Apple Intelligenceの設定、言語・地域、モデルの準備状態によって異なります。

● EPUB・PDF・HTMLへ書き出す

編集中のMarkdownを、明示的な操作でEPUB、PDF、HTMLへ書き出せます。EPUB書き出しは単一のMarkdown原稿を扱うための機能です。高度な書誌情報、表紙制作、目次編集、複数ファイルの書籍管理を備えたEPUB制作専用ツールではありません。

電子書籍モードは原稿を読み返すためのプレビューであり、書き出したEPUBと完全に同じ表示を保証するものではありません。最終的な表示はEPUB readerによって異なります。

● ローカルファースト

ログインやクラウド同期を前提にしません。ファイルはユーザーが選択したローカル環境で扱われます。Git、LSP、拡張機能、汎用ターミナル、任意コマンド実行をSafe Editorへ持ち込まず、文章を書くこと、読むこと、整えることに集中します。

Hazakura Editorは、Markdown原稿を自分の手元に置いたまま、本として読み、必要な箇所を直し、提案を確認して届けるための執筆環境です。

## What's New in 1.0.0

Hazakura Editor v1.0は、Markdownで書く、本として読む、ローカルAIで整える体験をひとつにまとめた最初の正式版です。

・L ModeとMarkdownプレビューで、sourceを保ったまま執筆できます。
・電子書籍モードと見開き表示で、長い原稿を本のように読み返せます。
・読んで気づいた場所から、対応するMarkdownへ戻って修正できます。
・Hazakura Local Assistの提案をDiffで確認し、採用または破棄できます。
・MarkdownをEPUB、PDF、HTMLへ明示的に書き出せます。
・ワークスペース、保存、Recovery、長文表示の安定性を整えました。

`1.0.0`は、App Storeで公開済みの`0.36.0`までに育ててきた機能を、ひとつの製品体験として伝え直すリリースです。

## Claim Boundary

- Local Assistはプレビューであり、対応環境でのみ利用できる。
- 外部AIサービスへのfallbackや自律的なagent動作はない。
- AI提案は自動保存せず、ユーザーがDiffで確認する。
- EPUB書き出しは単一Markdown向けで、総合的なEPUB制作環境ではない。
- App Store buildにAgent Workbench、CLI Agent、任意コマンド実行は含めない。
