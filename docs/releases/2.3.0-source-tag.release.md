# Hazakura Editor 2.3.0 Source / Local App Release

> **章立てレシピの持ち運びと、本全体 Reader の続きから再開**

Status: Published as a GitHub source / local-app tag on 2026-07-24. No binary
assets are attached. The Mac App Store listing for product line `2.3.0` was
reported published the same day (user-reported); that store fact is separate
from this source tag.

`v2.3.0` is the source / local-app checkpoint for portable Book recipe export /
import, whole-book Reader resume, Preview image and EPUB cover repairs,
recent-folder sandbox restore, and the folded v2.1–v2.2 quality pack (bounded
Reader search, chapter nav, export Finder reveal, Assist honesty, preflight
hints, e-book find-result sync). It publishes the repository state, source
archive, release notes, and build-from-source path. It does not attach an App
Store package or DMG.

Local App Store / TestFlight package provenance for the store build lives in
ignored `docs/internal/app-store-candidates/latest.json` and is not a GitHub
Release asset.

Prior source checkpoint remains immutable `v2.0.0` (Book Scope Alpha + Help).
Do not move that tag.

## Build From Source

```bash
npm ci
npm run build
```

The default local smoke bundle is generated at:

```txt
src-tauri/target/release/bundle/macos/Hazakura Editor.app
```

It is a local validation artifact, not an attached, notarized, or App
Store-distributed binary.

## What Changed Since 2.0.0

- **Portable Book recipe:** explicit export/import of relative-path JSON
  (`hazakura-book-recipe` v1). Import is always an editable draft until Save;
  never auto-loaded; not an OKF claim.
- **Reader resume:** app-private per-workspace chapter + scroll ratio for the
  whole-book Reader (max 8 workspaces); no project-file write.
- **Bounded whole-book search:** Reader searches loaded chapter names and
  visible Markdown under the existing 100-chapter / 32 MiB budget; `Command+F`
  / Enter navigation while Reader is open.
- **Preview image hardening:** near-viewport deferred load, nested WKWebView
  fallback, and data-URL commit so permitted images do not flash-then-blank.
- **Explicit EPUB cover:** optional local cover image per export; no first-image
  inference or source rewrite.
- **Recent folder sandbox restore:** per-workspace security-scoped bookmark
  with picker reauthorization fallback.
- **Quality pack:** Reader chapter prev/next, export Finder reveal, Book
  empty-state honesty, Assist availability honesty, preflight fix hints,
  e-book find-result sync into right-pane e-book and Reading Focus.
- Version metadata: npm, Tauri, Cargo → `2.3.0`.

## Release Boundary

v2.3.0 does not add:

- vertical writing (縦書き)
- cover cropping / cover editor
- auto-load recipes or workspace manifests
- background indexing, automatic repair, auto-apply, or a second editable buffer
- Agent Workbench / external CLI agents in the App Store lane
- network fallback, auto-save, tool calling, Git, LSP, terminal, or plugins
- binary assets on the GitHub tag

Markdown / text source remains canonical per file. Safe Editor remains primary.

## Known Limits

- Broad signed TestFlight interaction matrix, full VoiceOver, narrow-window,
  and long-document evidence remain follow-up rather than claims of this tag.
- Heavy-manuscript Apple Books / PDF visual proof may still deepen after
  publication; do not over-claim bulk export quality beyond exercised smoke.
- Residual polish (compact Book toolbar density, tab overflow, nav “back”,
  status TTL) may land on `main` after this tag and is not part of the tag claim.
- `cargo audit` may still report allowed unmaintained / platform transitive
  warnings triaged for prior releases; treat new high/critical findings as
  blockers if they appear on a future tag gate.
