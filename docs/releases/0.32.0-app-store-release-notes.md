## App Store Connect Copy

電子書籍モードと編集エリアの行き来を改善しました。

- 編集中の場所から電子書籍表示を開いたとき、近い章から読み始めやすくなりました。
- 電子書籍表示で読んでいる場所から、対応する編集位置へ戻りやすくなりました。
- 1ページ表示と「集中して読む」の見開き表示で、章やページ位置がそろいやすくなりました。
- 見開き表示では、章の境界で右ページに次の章をプレビューできる場合があります。
- 見出しを編集中に、位置同期で編集エリアが意図せず移動しないよう調整しました。
- 章末や見出しまわりで、不要な空白ページが出にくいよう調整しました。

## Longer Release Note

`0.32.0` improves the revision loop between Markdown editing and e-book
reading. The goal is to make "read, notice, return, fix" feel like one
continuous workflow while keeping Markdown source as the canonical
document.

Main changes:

- e-book Mode now opens near the current editor cursor or visible editor
  position when no fresher reader location exists.
- Returning from Reading Focus uses an approximate source-line estimate
  before falling back to the chapter heading.
- Right-pane one-page reader navigation can move the editor to the same
  chapter area without requiring Reading Focus first.
- The right-pane one-page reader and Reading Focus spread reader share
  reader location state, reducing drift between the two surfaces.
- Passive source edits and chapter reclassification no longer push the
  editor position back from the reader, avoiding jumps while editing
  headings.
- Spread view can use a spare right page at a chapter boundary to
  preview the next chapter while keeping the active location anchored to
  the current chapter.
- Pagination behavior around headings and chapter endings was adjusted
  to reduce unnecessary blank pages.

## Boundary

This release does not add a second document model, Preview DOM editing,
automatic rewriting, auto-save, Git integration, LSP, terminal,
arbitrary command execution, external AI/API calls, or Agent Workbench to
the App Store lane.

The reader/editor bridge uses approximate Markdown source positions, not
exact rendered-page coordinates. Normal saved documents, unsaved tabs,
and recovered dirty buffers should still be checked in the signed
TestFlight build before treating the candidate as App Review-ready.
