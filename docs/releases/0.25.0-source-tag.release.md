# Hazakura Editor 0.25.0 Source / Local App Tag

> **e-book reading surface and native-feeling macOS shell**

Status: Prepared for a GitHub source / local-app tag on 2026-06-19. No
binary assets are attached by this note.

`v0.25.0` is the source / local-app checkpoint for the e-book Mode
reading-surface work and the native-feeling macOS shell polish. It also
raises the local app's minimum macOS floor to macOS 26 so the next
native-material and App Store lanes can be evaluated honestly.

This release publishes the repository state, source archive, release
notes, and build-from-source path. It does not attach a DMG asset. The
latest downloadable warning-expected DMG preview remains `v0.20.0`.

The release boundary stays unchanged: Markdown source remains canonical,
Safe Editor remains primary, and Agent Workbench remains a separate
Developer / GitHub-lane trust boundary. The App Store lane remains
helper-free and omits Agent Workbench, CLI Agent launch, Apple Local
Assist helper, external AI/API calls, and arbitrary command execution
surfaces.

This is not an App Store, TestFlight, Developer ID signed, notarized,
installer, updater, or production-ready distribution.

## Build From Source

```bash
npm ci
npm run build
```

The local smoke bundle is generated at:

```txt
src-tauri/target/release/bundle/macos/Hazakura Editor.app
```

That local app is ad-hoc signed for local validation only. It is not
Developer ID signed or notarized.

## What Changed Since 0.20.0

### e-book Mode Reading Surface

- Adds the active-chapter e-book reader path and deepens it into a
  page-like single-page reading surface.
- Paginates the rendered active chapter body with CSS Columns while
  keeping reader chrome outside the column flow.
- Uses `column-fill: auto`, a fixed reader viewport, and chapter-local
  page state to avoid the empty-column trap and keep page movement
  predictable.
- Adds a fixed reader footer with chapter title and chapter-local
  `n / total` progress. Whole-book page numbering, EPUB export, vertical
  writing, and spread view remain deferred.
- Records the decision that right-pane 2-up is not a good fit; if
  spread reading returns, it should be part of a later e-book occupied
  mode rather than a right-pane toggle.

### Native-feeling macOS Shell

- Moves the app to overlay titlebar chrome with traffic-light-safe drag
  regions and no-drag controls.
- Tunes the native-feeling top chrome, tab row, sidebar, status bar, and
  right-pane mode controls after human-side visual checks.
- Adds native vibrancy support through the macOS-only `window-vibrancy`
  dependency and keeps dense editor / preview text on readable surfaces.
- Raises the minimum macOS floor to `26.0` for the local app metadata
  and Rust binary deployment target.
- Keeps all five themes available, with Sakura / Yakou / Shokou shell
  gradients used as visual accent rather than changing the saved
  document model.

### Release And Planning Alignment

- Aligns npm, Tauri, Cargo, and lockfile metadata to `0.25.0`.
- Marks the earlier local `0.20.0` App Store package candidate build
  `16` as superseded before upload. A fresh `0.25.0` App Store package
  must be generated before Transporter / App Store Connect work.
- Keeps AI Markdown ingest deferred to v0.26 so the shell and reading
  surface can settle first.

## Release Boundary

v0.25.0 does not add:

- App Store review approval, TestFlight distribution, or App Store
  submission proof.
- A new GitHub DMG asset.
- Developer ID notarization, stapling, auto-update, or installer-grade
  distribution.
- A full SwiftUI / AppKit rewrite or true Liquid Glass fidelity.
- Right-pane 2-up, EPUB export, vertical writing, whole-book page
  numbering, or a second saved document model.
- New Agent Workbench capability, provider plugins, command execution,
  terminal, LSP, Git integration, workspace-wide indexing, auto-apply,
  or auto-save.

## Known Limitations

- The minimum macOS floor is now `26.0`; older macOS versions are no
  longer supported by this source state.
- The latest downloadable warning-expected DMG preview remains
  `v0.20.0`; users who need a binary preview should use that release
  until a later DMG preview is published.
- The built local app is ad-hoc signed only and is not Developer ID
  signed or notarized.
- Native vibrancy and overlay titlebar behavior require built-app smoke
  on macOS 26; browser-only checks cannot prove window drag regions,
  traffic-light spacing, or native material rendering.
- App Store screenshots, Transporter upload, Apple validation,
  TestFlight smoke, App Review, and reviewer notes remain separate
  App Store-lane work.
