# hazakura editor 0.7.0 Warning-expected DMG Preview

> **Review Desk MVP And Release Prep**

v0.7.0 ships a small, honest **Review Desk MVP** on top of the existing non-Git diff and recovery review foundation.
It does not turn the editor into a Git client, merge tool, or general terminal.
Safe Editor Mode stays primary; Agent Workbench stays an explicit, allowlisted, one-session, no-restore, no-auto-apply trust boundary.

The full editable two-column Review Desk workbench, candidate temporary-document policy, bounded Global Search, Command Palette, settings consolidation, Frontmatter, KaTeX, and encoding conversion are deliberately deferred to v0.8+.
v0.7.0 keeps the lane narrow: explicit review, explicit apply, explicit safety checks.

This is a warning-expected macOS DMG preview. The app is ad-hoc signed and not notarized, so Gatekeeper warnings are expected.

Only use this preview if you are comfortable evaluating unsigned and not-notarized macOS preview binaries.

## What Changed Since 0.6.0

### Review Desk MVP

- Added a visible top-chrome `Review Desk` entry button and a `View > Review Desk` native menu item that toggle a dedicated review surface replacing the editor area.
- Reserved `Cmd+Shift+R` (and `Ctrl+Shift+R` on non-mac) as the Review Desk entry shortcut, with IME-composition and modal-open bail-out behavior.
- Added a manual candidate paste area inside Review Desk for comparing the active editor buffer against user-pasted candidate text.
- Added a localized `Compare` action that builds a real `CompareCase` (kind: `"candidate"`) and renders the diff preview by reusing the existing `DiffBody` view.
- Added an explicit `Apply candidate` action that only replaces the active editor buffer after a successful compare and never auto-saves the underlying file.
- Added Review Desk apply safety guards:
  - The preview is bound to the active tab id. If the active tab changes after compare, `Apply candidate` is disabled until a fresh compare runs for the new tab.
  - The preview is bound to the compared buffer snapshot. If the active buffer text changes after compare, `Apply candidate` is disabled until a fresh compare runs.
  - Editing the candidate textarea after a successful compare clears the preview, error, and apply action so the new text must be reviewed again.
  - A failed compare clears any stale successful preview before showing the error.
- Added `/review` and `/shortcut` slash commands and a `Shortcuts` slash category, with capture-phase `Enter` / `Tab` execution so the selected command runs without inserting a stray newline.
- Added a `Review Desk` status and entry copy group in English and Japanese, including `entryButton`, `entryButtonTitle`, and refreshed empty-state copy that points Diff / change-review users to the existing right-pane route.

### Diff / File Comparison Polish

- Tightened the dedicated Diff workbench setup pane to distinguish between no-workspace, source pending, target pending, same-file, and ready states, in both English and Japanese.
- Clarified the file-comparison column wording: `Close comparison result` / `比較結果を閉じる` for the result close action and `Set compare source` / `比較元に設定` for the context-menu side picker.
- Kept the GitHub-PR-like split Diff view and the right-pane compare route unchanged, and did not introduce Git-shaped labels (stage / commit / branch / merge / patch / apply).

### Theme And Menu Polish

- Adjusted Yakou / Shokou top chrome styling so open-file tabs, document meta, and the start surface follow the selected seasonal theme instead of falling back to OS-default-looking colors.
- Mapped Yakou to the native macOS dark window appearance and a transparent title bar background so the window title region no longer stays light while the app is in Yakou.
- Changed the native `View > Theme` submenu from independent check-style items to radio-style text markers (`●`) so the current theme reads as one selected choice.
- Expanded Japanese menu and dialog coverage for the release-prep surface.
- Added `かなふみ`, an intentionally hard-to-read, hiragana-heavy old-Japanese-ish menu language mode. In this mode the native menu bar changes to labels such as `ふみ`, `てならひ`, `ながめ`, `まど`, and `たすけ`.

### Stability, Safety, And Documentation

- The Review Desk keyboard-toggle close path resets manual candidate input, diff preview, and error before returning to the normal editor surface, matching the close-button reset behavior.
- App.tsx stays a pure composition file. New wiring routes through `useReviewDeskController`, `useReviewDeskState`, `useEditorCommands`, and `useSlashMenu` rather than expanding the composition file.
- Updated `docs/roadmap.md` to mark v0.7 as the Review Desk MVP and release-prep lane, with v0.8 owning the broader Review Desk UX workbench direction.
- Updated `docs/development-automation.md` to retarget the recurring automation guidance to v0.7 release-prep hardening, with editable two-column Review Desk, detached Agent windows, Global Search, Command Palette, Frontmatter, KaTeX, encoding conversion, and settings consolidation explicitly excluded from v0.7.
- Updated `docs/external-agent-review-workflow.md` to refocus the v0.7 review focus on boundary regressions, hidden Git / terminal / command behavior, and docs claims that exceed implementation.
- Updated `docs/current-status.md`, `docs/next-goals.md`, and `docs/smoke-checklist.md` so the documented Review Desk, manual candidate, and slash menu surface matches the implemented MVP.

## Boundary

v0.7.0 does not add:

- Git integration, merge editing, or advanced Git diff.
- LSP, general terminal, plugin system, or project-wide indexing.
- Arbitrary command execution, arbitrary command input UI, or arbitrary path input UI.
- Agent auto-apply, auto-commit, session restore, provider-add UI, or Pi RPC / SDK work.
- Editable two-column Review Desk, candidate temporary-document policy, detached Agent windows, or persistent review / session logs.
- Bounded Global Search, Command Palette, settings consolidation.
- Frontmatter, KaTeX, encoding display / Save As encoding, or theme editor.
- Signing, notarization, installer packaging, updater, or paid-distribution work.

Agent Workbench remains explicit, allowlisted (`codex` / `opencode` / `pi`), one active session, no restore, no auto-apply, and scoped to the selected workspace root. What happens inside the agent provider process is outside hazakura's safety boundary.

## Known Limitations

- This release does not add Git integration, LSP, project-wide indexing, arbitrary command execution, auto-apply, or auto-commit behavior.
- Agent Workbench provider behavior remains outside hazakura's safety boundary.
- Review Desk does not persist review / session logs, does not save candidate documents automatically, and does not auto-apply candidate text to the active file.
- Editable two-column candidate editing, candidate temporary-document policy, bounded Global Search, Command Palette, Frontmatter, KaTeX, encoding display, and settings consolidation remain v0.8+ work.
- The JavaScript bundle remains large; code splitting is planned for a future release.
- The bundled macOS binary is ad-hoc signed only and is not Developer ID signed or notarized.
