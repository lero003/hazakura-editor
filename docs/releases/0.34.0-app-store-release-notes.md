## Longer Release Note

`0.34.0` focuses on making PDF print usable in the Mac App Store /
TestFlight lane. The previous print path wrote temporary HTML and handed
it to the OS default browser. That was fragile under the App Sandbox, so
PDF print now stays inside the app boundary and uses an app-owned native
print webview.

Main changes:

- PDF print no longer depends on opening a temporary HTML file through
  the OS default browser.
- The app invokes the system print panel from an app-owned native print
  webview.
- Rust-side validation still requires the main window, non-empty HTML,
  and a plain `.html` file name.
- Stale print-browser handoff code and tests were removed.
- The v0.33 EPUB Export v1 Polish and small workspace / editor
  fit-and-finish items remain included in this candidate.

## Boundary

This release does not add a generic browser launcher, shell input,
Preview DOM editing, automatic rewriting, auto-save, Git integration,
LSP, terminal, arbitrary command execution, external AI/API calls, or
Agent Workbench to the App Store lane.

PDF print remains an explicit user action. Hazakura Local Assist remains
only an explicit, on-device, availability-gated writing companion whose
result stays unsaved until accepted and reviewable through Diff /
Discard.
