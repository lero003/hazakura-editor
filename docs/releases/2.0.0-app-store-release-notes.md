## App Store Connect Copy

Hazakura Editor v2.0は、選んだ Markdown を一冊として読み・書き出すための
「本」機能を追加し、知識フォルダ（OKF）との明示的な連携を整えたリリースです。

・サイドバーの「本」で、章にする Markdown を明示選択し、並べ順を保存できます（ディスクのファイル並びは変えません）。
・ワークスペースから章候補を作れます。index.md と、そこからつながる下位 index.md の章リンク順を優先し、保存するまで候補のままです。
・「本全体を読む」で、保存した章順の読み取り専用スクロール読書ができます。
・PDF / EPUB 書き出しで「現在のファイル」か「本全体」を選べます。本全体は章順と未保存タブを反映し、利用できない章があるときは書き出しを止めます。
・知識フォルダ（OKF）の点検とひな形作成はこれまで通り明示操作のみ。自動修正や起動時スキャンはありません。
・ヘルプに「本と知識フォルダ」を追加し、操作の要点だけを短く説明します。

Markdown sourceを正本とし、外部AIへのnetwork fallback、auto-save、tool calling、workspace-wide indexingを行わない境界は変わりません。

## Boundary

- Safe Editor remains primary; Markdown/text source remains canonical per file.
- Book order is app-private; not a workspace manifest and not OKF book semantics.
- No background indexing, auto-apply, auto-save, or second editable buffer.
- App Store lane omits Agent Workbench, CLI Agent, arbitrary command execution,
  and external AI/API providers.
- Hazakura Local Assist remains on-device, explicit, unsaved, and reviewable.

## Explicit Non-goals

- vertical writing (縦書き)
- whole-workspace Quick Open / background index
- Git, LSP, terminal, plugins, provider-add UI
- OKF auto-repair or silent multi-file rewrite

## Product Promise

Markdownで書き、本として読み、ローカルAIで整える — いまは「本」を明示選択した
ファイル群に広げられる段階です。説明は Help に置き、画面に機能を並べすぎません。
