## App Store Connect Copy

Hazakura Editor v1.12は、「書く・読む・確かめる」の流れと一つのMarkdown文書の構造確認を整え、知識フォルダをOKF v0.1 Draftの観点から点検し、よく使うMarkdown群のひな形から出発できるようにしたリリースです。

・Start Panel、ペイン表示、コマンドパレットを「書く・読む・確かめる」の言葉で整理しました。
・Outlineに見出し階層とページ区切りを表示し、構造上の気づきを修正を強制しない助言として示します。
・見出しレベルは、Undoできる明示的な単一文書の操作として変更できます。保存は引き続き明示操作です。
・コマンドパレットまたはフォルダメニューから「知識フォルダ（OKF）を点検」を実行できます。
・点検結果は「対応が必要」「OKFとして整える準備」「改善案」「参考情報」に分け、該当箇所を既存の編集タブで開いて修正後に再点検できます。
・「知識フォルダのひな形を作成（最小）」「知識フォルダのひな形を作成（本っぽい章立て）」で、上書きしない新しいフォルダに固定テンプレを一度だけ作成し、`index.md` を開きます。
・ひな形作成と点検はいずれも明示操作です。起動時の自動走査、バックグラウンド索引、複数ファイルの自動修正は行いません。

Markdown sourceを正本とし、外部AIへのnetwork fallback、auto-save、tool calling、workspace-wide indexingを行わない境界は変わりません。

## Release Position

`1.12.0` is the v1.12 OKF Starter Scaffold line. It carries v1.9 Writing
Loop Clarity, v1.10 Single-document Structure Foundation, v1.11 OKF Draft
Compatibility Preview, and the v1.12 explicit scaffold creation path.
It keeps Safe Editor invariants: no cloud OCR, no auto-save, no second
document model, and no Git / LSP / terminal / plugin expansion.

The product promise is unchanged:

> Markdownで書き、本として読み、ローカルAIで整える。

## Boundary

- Safe Editor remains primary; Markdown/text source remains canonical.
- OKF support is a compatibility preview against the pinned v0.1 Draft
  snapshot (`docs/okf-spec-pin.md`), not full conformance or Book Scope.
- Scaffold write and OKF review are separate explicit operations.
- App Store lane omits Agent Workbench, CLI Agent, arbitrary command
  execution, and external AI/API providers.
- Hazakura Local Assist remains on-device, explicit, unsaved, and reviewable.

## Explicit Non-goals

- Book Scope, chapter ordering, whole-book export
- Startup scan, persistent indexing, automatic multi-file repair
- Cloud OCR or external document upload
- Git, LSP, terminal, plugins, or arbitrary command execution
