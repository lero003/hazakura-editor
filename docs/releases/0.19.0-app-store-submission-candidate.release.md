# Hazakura Editor 0.19.0 App Store Submission Candidate

> **Helper-free App Store lane candidate**

Status: Prepared for local and TestFlight validation. Not submitted,
approved, or published.

`0.19.0` is the first App Store submission-candidate version after the
`v0.18.0` warning-expected Developer / GitHub DMG preview and the v0.18
pre-review quality closeout.

The release boundary stays unchanged: Markdown source remains canonical,
Safe Editor remains primary, and the App Store lane omits Agent
Workbench, CLI Agent launch, dev mode, arbitrary command execution,
Apple Local Assist helper, external AI/API calls, Git, LSP, terminal,
plugins, workspace-wide indexing, auto-apply, and auto-commit behavior.

## Candidate Identity

- User-visible version: `0.19.0`
- App Store build counter: `8`
- Bundle identifier: `dev.hazakura.editor`
- App name: `Hazakura Editor`
- Lane: helper-free App Store submission build

## What Carries Forward From 0.18.0

- Native macOS Trash path in the App Store lane, without `osascript` or
  AppleEvents.
- Bookmark-backed workspace restore regression coverage.
- Pasted-image decoded-size cap and image import safeguards.
- Direct save fallback failure recovery coverage.
- Status bar encoding / line-ending de-duplication.
- Bundled `LICENSE` and `THIRD_PARTY_NOTICES.md` resources.
- App Store surface omission checks for Agent Workbench, CLI Agent, and
  Apple Local Assist entry points.

## Required Evidence Before App Review Submission

- Build and verify the signed `0.19.0` App Store submit bundle.
- Package and upload `HazakuraEditor-0.19.0-build8-mas.pkg` through
  Transporter.
- Confirm App Store Connect processing / validation has no untriaged
  warnings.
- Run fuller TestFlight smoke on the resulting `0.19.0` build:
  workspace restore, outside-workspace active-tab restore,
  image paste/drag-drop, dirty `Cmd+Q` / red-close behavior,
  no external network communication, accessibility live checks, and
  App Store-lane Move to Trash.
- Finalize App Store Connect metadata, screenshots, support URL,
  reviewer notes, and privacy-policy URL.

## Local Smoke Snapshot

2026-06-12 local Codex smoke:

- `npm run build` produced the launchable helper-free local preview
  bundle at `src-tauri/target/release/bundle/macos/Hazakura Editor.app`.
- The local preview bundle reported version `0.19.0`, bundle identifier
  `dev.hazakura.editor`, executable `hazakura-editor`, and bundled
  `LICENSE` / `THIRD_PARTY_NOTICES.md`.
- The local preview bundle launched with `open -n`; the
  `hazakura-editor` process was observed and then stopped.
- `npm run build:app-store-submit` produced the universal App Store
  submit-lane bundle with `CFBundleShortVersionString=0.19.0` and
  `CFBundleVersion=8`.
- The submit-lane bundle had the expected App Sandbox,
  user-selected read/write, app-scoped bookmark, and network-client
  entitlements, and omitted the Apple Local Assist helper.
- The submit-lane bundle was signed with the Apple Distribution identity
  during local packaging, and the `.pkg` was signed with the 3rd Party
  Mac Developer Installer identity.
- Local Gatekeeper assessment can still report `Insufficient Context`
  for this lane. Treat launch validation as signed TestFlight proof, not
  as proven by local package inspection alone.

## Human-Side Smoke Snapshot

2026-06-12 human-side App Store lane smoke:

- Passed: first launch, new Markdown creation, opening an existing
  Markdown file through user selection, Preview / HTML export, sandboxed
  image paste / drag/drop, App Store surface omission, and dirty-tab
  `Cmd+Q` / red-window-button close confirmation.
- Partial: Save / Save As did not show data loss, but the Save As tab
  transition felt surprising enough to recheck before treating the UX as
  final.
- Acceptable with residual risk: workspace restore did not appear broken
  in follow-up smoke, but Google Drive-backed folders and
  quit-before-interaction timing should stay under observation on the
  exact signed TestFlight build.
- Passed enough for the candidate: Move to Trash appeared normal in the
  tested lane.
- Passed enough for the candidate: network observation did not show a
  concrete external app-data issue in the tested flow.
- Partial: VoiceOver was partly usable, but full live VoiceOver,
  keyboard-only traversal, and Increase Contrast checks remain
  incomplete.
- Code-level follow-up: `Cmd+Shift+F` global search returned results,
  but activating a result did not open the file. This is fixed after the
  human smoke note by routing result activation to the existing
  workspace-file opener and line jump handler.

## Not Claimed

This candidate is not yet:

- submitted to App Review,
- approved by App Review,
- a Developer ID signed or notarized DMG,
- a public GitHub binary release, or
- a `1.0.0` product milestone.

Apple Local Assist remains outside this App Store submission candidate.
The `1.0.0` milestone is intentionally reserved for a later product
decision after the App Store Safe Editor lane is proven.
