## App Store Connect Copy

Hazakura Editor v1.8は、Reference Compareと日々の執筆を、長い原稿やキーボード・VoiceOver利用時にもより確実に扱えるよう整えたリリースです。

・L Modeでは参照内容を保持したまま編集面に集中し、通常編集へ戻ると同じ参照を再表示します。
・長いテキスト参照の折返し、スクロール、選択、コピーを安定させ、上限を超える資料は編集中の文章を変えずに理由を示します。
・Quick Open、コマンドパレット、ファイル内検索、タブ、ファイルツリー、参照ペインのキーボード操作と読み上げ用の名前を改善しました。
・日本語とかな表示の案内を整え、PDF/EPUB書き出し前に未保存の変更、画像の扱い、保存先選択の流れを確認できます。
・検索失敗や参照PDFの再読込など、失敗時の状態をより明確に表示します。
・PDF書き出しの背景と長文末尾、テーマ描画負荷を安定させました。
・YAML frontmatter内のメモをOutlineの見出しとして誤認しないよう整えました。

Markdown sourceを正本とし、外部AIへのnetwork fallback、auto-save、tool calling、workspace-wide indexingを行わない境界は変わりません。

## Release Position

`1.8.0` is the v1.8 Daily Trust Completion published release. It hardens the
v1.7 Reference Compare plus trust / daily-use experience: L Mode keeps a
loaded Reference session while hidden and restores it on return; long text
references render wrap-safe inside a bounded DOM budget; keyboard and
screen-reader semantics, kana UI copy, export preflight, theme cost, and
failure-state messaging are tightened. It keeps Safe Editor invariants: no
cloud OCR, no auto-save, no second document model, and no Git / LSP /
terminal / plugin expansion.

The product promise is unchanged:

> Markdownで書き、本として読み、ローカルAIで整える。

## Boundary

- Safe Editor remains primary; Markdown/text source remains canonical.
- App Store lane omits Agent Workbench, CLI Agent, arbitrary command
  execution, and external AI/API providers.
- Hazakura Local Assist remains on-device, explicit, unsaved, and reviewable.
- One editable Markdown document may be shown beside one read-only reference;
  this is not a generic split editor or second document model.
- Extended TestFlight interaction breadth, spoken VoiceOver, and the full
  accessibility matrix remain v1.8 follow-up verification rather than
  publication blockers or reasons to mutate the released tag.

## Explicit Non-goals

- Cloud OCR or external document upload
- Auto-save or auto-apply of import or assist results
- Two editable panes, Review Desk revival, or Book Scope
- Git, LSP, terminal, plugins, or arbitrary command execution
