## App Store Connect Copy

Hazakura Editor v2.1では、「本全体を読む」画面に本の中を検索する機能を
追加しました。

・目次から、読み込んだ章のファイル名と本文を検索できます。
・一致した章と件数を表示し、結果から該当する章へ移動できます。
・開いている章では未保存の編集内容も検索対象になります。
・Escキーは入力中の検索語を先に消し、もう一度押すとReaderを閉じます。
・検索は「本全体を読む」を開いた時に読み込んだ範囲だけで行い、バックグラウンド索引や永続データを作りません。

Markdown sourceを正本とし、検索によるsource変更、auto-save、background
indexing、workspace-wide scanを行わない境界は変わりません。

## Candidate Boundary

- Search is limited to the explicit whole-book Reader load: at most the
  existing 100 chapters and 32 MiB total budget.
- Closed leading YAML frontmatter is excluded because Reader does not render it.
- Matching is Unicode-normalized and case-insensitive; it does not interpret
  Markdown semantics or create a persistent index.
- Narrow-window Reader keeps the bounded contents/search area available.
- App Store lane still omits Agent Workbench, CLI Agent, arbitrary command
  execution, and external AI/API providers.
