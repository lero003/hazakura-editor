・PDFや画像を、オンデバイスのPDFKit / VisionでMarkdown下書きとして取り込めるようになりました。結果は未保存の編集バッファとして開き、内容を確認・修正してから保存できます。

・ファイルメニュー、コマンドパレット、ワークスペースの右クリックから取り込みを開始できます。
・お遊びテーマ「江戸彼岸」を追加しました。Markdownのsource契約やSafe Editor境界は変わりません。
・PDF書き出しやローカル画像パスの取り扱いを整え、プレビュー / HTML / PDFで同じワークスペース境界を共有するようにしました。
・Markdownで書き、本として読み、ローカルAIで整える体験は、引き続きsourceを保ったまま動きます。

## Release Position

`1.6.0` is the v1.6 Import Assist Phase 1 release. It adds on-device PDF /
image intake into an unsaved Markdown draft while keeping Safe Editor
invariants: no cloud OCR, no auto-save, no second document model, and no
Git / LSP / terminal / plugin expansion.

The product promise is unchanged:

> Markdownで書き、本として読み、ローカルAIで整える。

The core flow remains write -> read -> notice -> return -> revise ->
review -> export. Import Assist adds an intake path into that flow:
import -> edit draft -> save explicitly.

## What Changed In v1.6

### User-visible improvements

- **Import Assist Phase 1.** Explicit PDF / image intake produces an
  unsaved Markdown draft. PDFKit text extraction is preferred; pure-Rust
  fallback and page OCR cover weaker text layers and scans. Image intake
  uses Vision OCR. No cloud path and no auto-save.
- **Discoverability.** File menu, command palette, and workspace
  right-click can start import with a confirmation step.
- **江戸彼岸 theme.** Presentation-only joke theme added after the closed
  v1.5 line; not part of the v1.5 release story.
- **PDF / image path trust.** Preview, HTML export, and PDF export share
  document-relative, workspace-contained image resolution. Parent workspace
  that owns manuscript and assets is the supported layout.
- **Export reliability polish.** PDF last-line safety, longer export
  timeout budget, and related layout guards that shipped with the 1.6
  quality packs.

### Trust / maintenance

- Nested App Store helpers (Local Assist + Import Assist) inherit re-sign
  in the submit lane.
- CodeMirror `@codemirror/view` remains pinned at **6.43.2** for display
  stability on long Japanese Markdown.
- Pure-Rust PDF text fallback uses `pdf-extract` 0.12.0 / `lopdf` 0.42.0
  (clears the vulnerable `lopdf` 0.34 path reported by RUSTSEC-2026-0187).

## Explicit Non-goals (unchanged)

- Cloud OCR or external document upload
- Auto-save or auto-apply of import results
- Embedded PDF image extraction into workspace assets (deferred)
- Book Project / multi-file chapter split (v2)
- Side-by-side PDF/Markdown reference compare (v1.7)
- Git, LSP, terminal, plugins, or arbitrary command execution
