# Hazakura Editor 2.0.0 Source / Local App Release

> **選んだ Markdown を一冊として読み・書き出す Book Scope と、本・知識フォルダの Help**

Status: Published as a GitHub source / local-app tag on 2026-07-21. No binary
assets are attached. The Mac App Store listing for product line `2.0.0` was
reported published the same day (user-reported); that store fact is separate
from this source tag.

`v2.0.0` is the source / local-app checkpoint for Book Scope Alpha, whole-book
read/export, UX quieting, and in-app Help for books and knowledge folders
(OKF). It publishes the repository state, source archive, release notes, and
build-from-source path. It does not attach an App Store package or DMG.

Local App Store / TestFlight package provenance for the store build lives in
ignored `docs/internal/app-store-candidates/latest.json` and is not a GitHub
Release asset.

## Build From Source

```bash
npm ci
npm run build
```

The default local smoke bundle is generated at:

```txt
src-tauri/target/release/bundle/macos/Hazakura Editor.app
```

It is a local validation artifact, not an attached, notarized, or App
Store-distributed binary.

## What Changed Since 1.14.0

- **Book sidebar:** Files / Book switch; explicit Markdown chapter selection
  (bounded); app-private ordered document/group tree; same-parent reordering;
  unavailable-entry retention and recheck; one primary edit buffer.
- **Chapter suggestion draft:** On-demand, bounded OKF snapshot adapts index
  headings and safe relative / bundle-root links into an editable draft tree;
  save remains explicit; no OKF version/type in the saved Book tree; no
  startup or background scan.
- **Whole-book reader:** Explicit read-only scroll reader in saved order; live
  dirty tabs win over disk; per-chapter image/link bases; partial-load notices.
- **PDF / EPUB export:** Current file or whole book; ordered multi-chapter
  output; bounded preflight; EPUB uses saved tree for navigation and rewrites
  links between included chapters to packaged XHTML targets.
- **UX quieting:** Settled Book view keeps read/edit primary; suggest and
  recheck stay progressive; user-facing copy avoids heavy “Book Scope” jargon.
- **Help:** Native Help menu / Command Palette → **Books and knowledge
  folders…**; About / diagnostics derive package version `2.0.0`.
- Version metadata: npm, Tauri, Cargo → `2.0.0`.

## Release Boundary

v2.0.0 does not add:

- vertical writing (縦書き)
- background indexing, automatic repair, auto-apply, or a second editable buffer
- workspace manifest as truth for chapter order (order stays app-private)
- Agent Workbench / external CLI agents in the App Store lane
- network fallback, auto-save, tool calling, Git, LSP, terminal, or plugins
- binary assets on the GitHub tag

Markdown / text source remains canonical per file. Safe Editor remains primary.

## Known Limits

- Broad signed TestFlight interaction matrix, full VoiceOver, narrow-window,
  and long-document evidence remain follow-up rather than claims of this tag.
- Heavy-manuscript Apple Books / PDF visual proof may still deepen after
  publication; do not over-claim bulk export quality beyond exercised smoke.
- `cargo audit` may still report allowed unmaintained / platform transitive
  warnings triaged for prior releases; treat new high/critical findings as
  blockers if they appear on a future tag gate.
- Residual polish (tab overflow, nav “back”, status TTL) stays parked.
