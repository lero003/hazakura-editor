## Release Position

`1.7.0` is the v1.7 Reference Compare plus trust / daily-use hardening
TestFlight candidate. It generalizes the post-import review need into one safe
model: editable Markdown stays center / primary; one read-only PDF, image,
Markdown, or text reference opens on the right as a preview-like pane (not
Diff). It keeps Safe Editor invariants: no cloud OCR, no auto-save, no second
document model, and no Git / LSP / terminal / plugin expansion.

The product promise is unchanged:

> Markdownで書き、本として読み、ローカルAIで整える。

The core flow remains write -> read -> notice -> return -> revise ->
review -> export. Reference Compare adds a beside-editor reference into that
flow: open reference -> compare while editing -> save explicitly.

## What Changed in v1.7

### User-visible improvements

- **Reference Compare.** One editable Markdown tab stays center / primary;
  one read-only PDF, image, Markdown, or text reference opens on the right.
  Not Diff. Import Assist pairs its source (right) with the unsaved draft
  (center); workspace / external files may also open as a right-hand reference
  while editing.
- **PDF reference reader.** Multi-page PDF references render actual page
  images on the right with next / previous page, fit width, fit page, and zoom
  controls. Pages use CSP-compatible bounded `data:image` rendering.
- **Page-level advisory navigation.** Import Assist pairs show page-level
  `要確認` advisory markers from OCR confidence. Advisory only — no
  per-character confidence claims. Manual pause / resume / follow controls.
- **L Mode continuity.** L Mode is Markdown-document-only, so CSS / HTML / XML
  documents no longer force a Markdown parser remount that would lose Undo
  history. Side-pane state and editing context stay more stable across the
  L Mode toggle.
- **Explicit recovery for pathless drafts.** New-file and Import Assist drafts
  are kept in a separate app-private recovery store, bounded by size and TTL.
  Recovery is always shown as an explicit candidate that opens a new pathless
  tab; it never silently overwrites a saved source file.
- **Long-reference rendering.** Long text references render with natural-height
  wrapping, full selection, and scroll inside a separate 1.5M-character /
  50,000-line budget. Over-budget input is rejected with a visible reason
  before replacing an already-open reference.

### Trust / maintenance

- Recovery cleanup failures are now surfaced across Save / Save As / restore /
  discard / close without blocking editing or an explicit close.
- Text-reference over-budget rejection keeps the editable buffer unchanged.
- CodeMirror `@codemirror/view` remains pinned at **6.43.2** for display
  stability on long Japanese Markdown.
- Reference Compare is not Diff, not two editable panes, and not a second
  document model.

## Remaining Verification Before Publication

`1.7.0` has user-reported basic packaged smoke with no issue, but it is not a
published release. The following remain useful extended verification and
post-submission observation rather than claims already completed:

- Extended packaged Reference Compare interaction matrix (PDF / image / text references,
  Import Assist pair, page follow, Save As / tab switch / dirty close /
  reference replacement, narrow window, keyboard-only, App Store sandbox
  picker).
- Forced-termination recovery smoke for pathless drafts.
- Long-text-reference wrap / scroll / selection near the budget boundary.
- Accessibility matrix (keyboard / VoiceOver / contrast / enlarged text /
  Reduce Motion).

## Explicit Non-goals (unchanged)

- Cloud OCR or external document upload
- Auto-save or auto-apply of import or assist results
- Two editable panes or generic split-editor behavior
- Review Desk revival
- Book Project / multi-file chapter split (v2)
- Git, LSP, terminal, plugins, or arbitrary command execution
