## Longer Release Note

`0.33.0` polishes the initial EPUB export for single-document book
writing and adds two small workspace / editor fit-and-finish items. The
goal is to make EPUB export feel like a normal v1 workflow rather than a
beta experiment, while keeping Markdown source as the canonical document.

Main changes:

- EPUB export is presented as `EPUB書き出し` / `EPUB Export` instead of
  beta copy. It remains an explicit active-document action over Markdown
  source.
- Successful exports that replaced some images now report a warning
  status rather than a silent success. The first report type is
  `image-unavailable`.
- Generated EPUB nav/content XHTML uses the language metadata selected
  in the export dialog instead of hardcoding `ja`.
- The workspace tree shows existing-tab-derived open and dirty markers
  for files inside the selected workspace, reusing the tab dirty state
  so unsaved content, line-ending, and encoding changes stay aligned
  with the tab bar.
- The editor content area opens the existing slash-command menu from a
  right-click; it preserves selection when invoked inside the selection,
  otherwise moves the cursor to the clicked position.
- E-book page navigation and Preview / e-book display styling were
  refined for the v1 reading and revision flow.

## Boundary

This release does not add a second document model, Preview DOM editing,
automatic rewriting, auto-save, Git integration, LSP, terminal,
arbitrary command execution, external AI/API calls, or Agent Workbench to
the App Store lane.

EPUB export remains an explicit user action. It writes a minimal EPUB
archive from the active Markdown source; it does not guarantee
reader-perfect EPUB reproduction against the e-book Mode preview, does
not add a cover editor or advanced metadata, and keeps EPUBCheck as a
manual outside-app step. Hazakura Local Assist remains only an explicit,
on-device, availability-gated writing companion whose result stays
unsaved until accepted and reviewable through Diff / Discard.
