# hazakura-note 0.1.0 Source-only Developer Preview

`hazakura-note` 0.1.0 is a source-only developer preview for reviewing and building the current Markdown-first safe editor from source.

This release does not include a signed, notarized, or installer-grade macOS binary asset.

## Install And Build From Source

```bash
npm ci
npm run build
```

The local macOS app bundle is generated at:

```txt
src-tauri/target/release/bundle/macos/hazakura-note.app
```

For development runs:

```bash
npm ci
npm run dev
```

Use `npm install` only when intentionally updating dependencies and the lockfile.

## Included

- Tauri + React + CodeMirror Markdown/text editing prototype.
- Local file open, edit, save, Save As, and sanitized Markdown preview.
- Bounded workspace tree and multiple tabs.
- Native File and View menus.
- Save conflict and save failure recovery flows.
- Explicit unsaved draft recovery after restart.
- Workspace image preview for local PNG/JPEG/GIF/WebP files up to 20 MB.

## Not Included

- Binary distribution.
- Developer ID signing.
- Notarization.
- Installer packaging.
- Auto-update.
- Git integration.
- LSP.
- Terminal.
- Plugin system.
- Arbitrary command execution.
- AI rewrite or completion.
- Project-wide indexing.

The generated local `.app` is ad-hoc signed for local build validation only. It is not Developer ID signed or notarized.

## Local Verification

```bash
npm ci
npm run build:vite
cargo fmt --manifest-path src-tauri/Cargo.toml -- --check
cargo test --manifest-path src-tauri/Cargo.toml
npm run build
git diff --check
npm audit
cargo audit
```

`cargo audit` currently completes with allowed warnings from transitive Tauri/wry dependencies, mostly GTK3/gtk-rs Linux stack advisories plus `unic-*` and `glib` warnings. These are tracked for dependency-update review and are not treated as macOS source-preview blockers.

## Known Limits

- The file tree is a workspace browser, not a project index.
- Save conflicts are recoverable, but there is no merge editor.
- Unsaved draft recovery is explicit and fingerprint-bound; it is not autosave.
- The production bundle currently reports a Vite chunk-size warning from editor and preview dependencies.
- Some recent polish paths should receive broader built-app smoke before being treated as distribution-grade, especially dirty close failure/conflict handling, Discard All draft cleanup, Find highlight cleanup, editor keyboard handling, workspace image preview, and Markdown image blocking.

## Publication Boundary

Do not attach a macOS binary asset to this source-only release.

If a warning-expected DMG preview is approved later, use `docs/dmg-preview-checklist.md` and publish it as a distinct preview lane rather than silently changing this source-only release.
