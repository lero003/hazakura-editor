## Longer Release Note

`0.36.0` focuses on the e-book reader page-turn experience. Long,
illustrated documents could skip pages (or whole chapters) while
flipping, and the chapter structure fragmented heavily when a document
used many `###` subheadings. This release realigns the reader with an
EPUB-like page-break rule.

Main changes:

- The chapter-cross page target is recomputed so one user action moves
  one spread and crosses at most one chapter boundary, landing on the
  chapter opener for unseen chapters and on the continuation for
  chapters already previewed on the spare right spread page.
- Keyboard auto-repeat (`event.repeat`) is ignored so a held key does
  not outrun the page-state update and skip pages.
- A pending chapter-cross target guards rapid turns while the next
  chapter's HTML is still rendering / measuring.
- Reader chapters now treat only H1 (`#`) and H2 (`##`) as boundaries;
  H3 and below stay inside the preceding chapter as in-chapter
  subheadings. The editor ↔ reader position bridge uses the same
  coalescing so chapter indices stay aligned.
- Page counts measured while images are still decoding are held
  provisional and never shrink the committed count within a chapter, so
  the active page no longer jumps backward as images load.
- EPUB export splits the body into separate XHTML content documents at
  explicit page-break markers (`---` / `===`) and keeps the OPF spine
  and navigation in sync, instead of relying on CSS page breaks.

## The v1 Book-Writing Arc

Hazakura Editor is becoming a safe Markdown book editor: you write
Markdown, read it like a book, notice something, return to the matching
source location, revise, review an AI proposal through Diff, and
export. The e-book reader page-turn stabilization in this release is
part of making "read → notice → return" feel like one revision cycle
rather than a separate viewer.

## Boundary

This release does not add a generic browser launcher, shell input,
Preview DOM editing, automatic rewriting, auto-save, Git integration,
LSP, terminal, arbitrary command execution, external AI/API calls, or
Agent Workbench to the App Store lane.

The e-book reader and EPUB export remain explicit, on-device surfaces.
Markdown source remains canonical.
