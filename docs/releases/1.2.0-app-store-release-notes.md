## Release Position

`1.2.0` is the v1.2 Polish And Expectation Setting release. It does not
add a new product surface, a second document model, arbitrary command
execution, an external AI/API path, or a broader workspace model. It
deepens the existing single-document Safe Editor with safer command
discovery and clearer export expectations.

The product promise is unchanged:

> Markdownで書き、本として読み、ローカルAIで整える。

The core flow remains write -> read -> notice -> return -> revise ->
review -> export. Markdown source remains canonical throughout that flow.

## What Changed In v1.2

- **Safe command discovery.** The command palette (`Cmd+Shift+P`,
  `useCommandPaletteController.ts`) now exposes the same safe Markdown
  actions as the right-click slash menu — headings, bullet / numbered /
  task lists, quote, code block, divider, image, strikethrough, plus the
  existing bold / italic / inline-code / link / 3-column table / date /
  time actions. Palette execution is derived from the existing
  slash-command registry, so insert text and action callbacks stay
  shared; table insertion preserves a selected range instead of
  replacing it.
- **Localized search vocabulary.** Slash labels and search keys are
  merged into palette keywords, so Japanese / kana terms (for example
  `見出し` / `みだし`) remain searchable while palette labels stay English.
  Full palette i18n is a separate future slice.
- **Context-menu viewport containment.** The right-click slash menu now
  opens upward near the lower edge, aligns from the right near the right
  edge, clamps the left edge to an 8px viewport inset, and bounds its
  max dimensions to the viewport.
- **EPUB export scope note.** The EPUB settings dialog now carries an
  in-app scope note stating that e-book Mode is a reading preview, the
  exported EPUB targets a single Markdown document, and Hazakura is not
  a full EPUB production tool (Japanese and English).

## Boundary

Hazakura Local Assist remains a preview, availability-gated on-device
writing companion. It has no external AI/API fallback, background
rewriting, auto-save, tool calling, or workspace-wide indexing. Its
proposals remain explicit, unsaved, and Diff / Discard reviewable.

EPUB export remains an explicit single-document workflow. `1.2.0` does
not claim advanced metadata, cover design, navigation editing, multi-file
book management, or a full EPUB production environment. e-book Mode is a
reading preview, not a guarantee that every EPUB reader will reproduce
the same layout.

The App Store lane excludes Agent Workbench, CLI Agent launch, arbitrary
command execution, external AI/API calls, and provider-add UI. The
command palette and right-click slash menu expose only the existing
allowlisted Markdown insertion / wrapper actions; they do not add
arbitrary command execution, a formatting toolbar, Git status,
background indexing, or a new Agent / Review surface.

reproducible or indicate a hotfix risk.
