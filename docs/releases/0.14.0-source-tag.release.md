# hazakura editor 0.14.0 Source / Local App Tag

> **L Mode Stability Ramp / Hazakura Local Assist Harness Polish**

v0.14.0 records the first focused L Mode stability pass plus a bounded Hazakura Local Assist harness polish pass as a source / local-app checkpoint.

This tag is a source / local-app checkpoint. It is not a signed, notarized, App Store, TestFlight, or warning-expected DMG publication.

The latest published downloadable DMG preview remains `v0.11.0`.

## What Changed Since 0.13.0

### L Mode Stability

- Tightens the L Mode decoration recompute trigger by structurally comparing `transaction.startState.selection` with `transaction.newSelection`, so real selection movement is covered while same-selection redispatches stay cheap.
- Adds a two-stage Typewriter / IME guard: the plugin skips measured recentering while `view.composing` is true at update entry and re-checks composition state inside the deferred rAF callback.
- Adds visual-overlap fixtures for L Mode content width, page padding, margin-chip geometry, and narrow / wide chip-to-padding headroom.
- Makes L Mode task widgets keyboard focusable and toggles `[ ]` / `[x]` on Enter or Space while preserving Markdown source as the saved text.
- Adds a screen-print-only L Mode fallback that hides floating chrome, removes dim / chip styling, unwinds the centered column, and returns the surface to plain white when printing the active editor screen.

### Hazakura Local Assist Harness Polish

- Builds Hazakura Local Assist `documentContext` around the active target instead of blindly sending the document head, so long documents give the model nearby section context.
- Clamps the actual returned context slice and snaps to safe line boundaries without erasing the lines around the target.
- Keeps the L Mode floating review sheet from inheriting the normal diff row's fixed `min-width: 720px`, reducing horizontal scroll for long one-line Assist candidates.
- Expands Hazakura Local Assist window copy and classifies common raw apply errors into localized user-facing messages for selection-too-long, context-too-long, stale-target, guardrail, throttled, and unknown-error paths.

### Boundary Clarification

- Keeps Markdown source canonical.
- Keeps L Mode rendering inside CodeMirror decorations.
- Keeps Preview DOM editing, `contenteditable`, HTML as the saved canonical model, and hidden save-time rewriting out of scope.
- Keeps the canonical Print to PDF / Export HTML path in `useDocumentExport`, which renders Markdown source through the standalone preview/export pipeline. The L Mode `@media print` block is only a screen-print fallback and cannot CSS-recover source text from `Decoration.replace` widgets.

## Known Limitations

- Built-app manual smoke for real Japanese IME candidate-window behavior remains useful when practical; unit tests pin the `view.composing` guard and rAF race guard but do not prove the native candidate-window feel.
- The visual-overlap fixture is numerical CSS evidence, not a screenshot / WebView geometry proof for every theme, heading, blockquote, fenced code block, or long Japanese heading.
- The L Mode screen-print fallback is not the canonical export path. Users who need source-derived HTML or PDF output should use the existing Export HTML / Print to PDF flow.
- `Decoration.replace` widgets such as Image, HorizontalRule, TaskMarker, and TableDelimiter cannot be restored to original source text by CSS during screen print.
- Hazakura Local Assist remains alpha / experimental and depends on macOS 26+ Apple Foundation Models availability, local Apple Intelligence state, and supported language / locale.
- The local `.app` produced by `npm run build` is ad-hoc signed only and is not Developer ID signed or notarized.

## Local Verification Before Tagging

Required gates for this source / local-app tag:

```bash
npm ci
npm run typecheck
npm run test
npm run build:vite
cargo fmt --manifest-path src-tauri/Cargo.toml -- --check
cargo test --manifest-path src-tauri/Cargo.toml
npm run build
git diff --check
npm audit
cargo audit --file src-tauri/Cargo.lock
```

Review-only dependency checks:

```bash
npm outdated
cargo update --manifest-path src-tauri/Cargo.toml --dry-run
```

The built app must report version `0.14.0`, bundle identifier `lab.hazakura.note`, product name `hazakura editor`, and executable `hazakura-editor`.

The built app must pass:

```bash
codesign --verify --deep --strict --verbose=2 "src-tauri/target/release/bundle/macos/hazakura editor.app"
```

Expected warning for the local ad-hoc app:

```bash
spctl -a -vv -t open "src-tauri/target/release/bundle/macos/hazakura editor.app"
```

Gatekeeper may reject the app with insufficient-context / not-notarized style output because this tag does not produce a notarized distribution asset.
