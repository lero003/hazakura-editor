# Hazakura Editor 0.34.0 App Store / TestFlight Candidate

> **Helper-enabled App Store lane package candidate**

Status: local package evidence generated on 2026-06-25, then superseded
for PDF output after TestFlight still showed macOS' "This application
does not support printing" alert. App Store Connect upload, processing,
TestFlight install / launch, App Review, and release handling are
performed outside this repository unless separately recorded.

`0.34.0` packages the native PDF print fix on top of the v0.33 EPUB
Export v1 Polish, v1 workspace marker, right-click slash-command
fit-and-finish, and v1.0 RC preparation work. The release boundary stays
unchanged: Markdown source remains canonical, Safe Editor remains
primary, and the App Store lane omits Agent Workbench, CLI Agent launch,
dev mode, arbitrary command execution, external AI/API calls, Git, LSP,
terminal, plugins, workspace-wide indexing, auto-apply, and auto-commit
behavior. Hazakura Local Assist remains only an explicit, on-device
writing companion whose final result enters the existing unsaved edit
transaction / Diff review flow.

## v0.34 Candidate Focus

- PDF print no longer depends on opening a temporary HTML file through
  the OS default browser.
- The app creates an app-owned native print webview and invokes the
  system print panel from inside the app boundary.
- Rust-side guards remain in place for the main window label,
  non-empty HTML content, and plain `.html` file names.
- The old print-browser handoff command path and stale tests were
  removed.
- The user reported the manual native PDF print flow as working before
  this package candidate was generated.
