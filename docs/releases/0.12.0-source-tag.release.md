# hazakura editor 0.12.0 Source / Local App Tag

> **Hazakura Local Assist Alpha**

v0.12.0 adds Hazakura Local Assist as an alpha / experimental Assist Surface for lightweight on-device text assistance. It is designed for short summaries, rephrasing, heading / tag ideas, light cleanup, and small writing edits that remain explicit, unsaved, and reviewable through Diff before the user saves.

This tag is a source / local-app checkpoint. It is not a signed, notarized, App Store, TestFlight, or warning-expected DMG publication.

The latest published downloadable DMG preview remains `v0.11.0`.

## What Changed Since 0.11.0

### Hazakura Local Assist Alpha

- Adds the external Hazakura Local Assist companion window as the `apple-local` Assist Surface provider.
- Lets Hazakura Local Assist and the Agent Window share the outside companion slot: the app normally shows one or the other, not both as active companions.
- Supports rough writing requests from normal editor and L Mode.
- Uses the bundled Swift helper to call Apple Foundation Models when available on the current Mac.
- Keeps availability runtime-gated: unavailable, unsupported, disabled, language-limited, and long-running states are surfaced to the user instead of falling back to a network model or hidden fixture.
- Records direct AI-written buffer changes as unsaved AI edit transactions with before/after diff review and discard affordances.
- Keeps Hazakura Local Assist labeled as alpha / experimental and as lightweight text assistance, not as the main AI feature.

### Assist Surface Settings

- Adds Assist Surface selection for `Hazakura Local Assist (Experimental)` / `CLI Agent` / `Off`.
- Keeps CLI Agent under the existing Agent Workbench boundary: restart-required enablement, allowlisted providers, selected workspace, and responsibility consent.
- Keeps Hazakura Local Assist separate from CLI agents and provider configuration.
- Renames visible Apple assist labels toward `Hazakura Local Assist` to avoid presenting it as a broad AI assistant.

### Release Boundary

v0.12.0 does not add:

- App Store or TestFlight submission.
- Developer ID signing, notarization, installer packaging, updater, or distribution-grade Gatekeeper approval.
- A warning-expected DMG asset for this tag.
- Network-backed LLM fallback.
- External AI-agent replacement, local LLM runtime replacement, or generic chat.
- Code review, multi-file understanding, long-document restructuring, autonomous agent work, tool calling, workspace indexing, or command execution.
- Auto-save, background rewriting, hidden apply, or irreversible AI edits.

## Known Limitations

- Hazakura Local Assist live generation depends on macOS 26+ Apple Foundation Models availability, local Apple Intelligence state, and a Foundation Models-supported current app language / locale.
- Output quality is alpha-level and may be weak for real writing tasks.
- Hazakura Local Assist is not suitable for code review, multi-file understanding, long-document restructuring, advanced reasoning, or external-agent replacement.
- The editor-wide `minimumSystemVersion` remains `11.0`; older Macs can run the editor while Hazakura Local Assist reports unavailable or unsupported.
- The JavaScript bundle remains large; code splitting is still future hardening work.
- The local `.app` produced by `npm run build` is ad-hoc signed only and is not Developer ID signed or notarized.

## Local Verification Before Tagging

Required gates for this source / local-app tag:

```bash
npm run typecheck
npm run test
npm run build:vite
cargo fmt --manifest-path src-tauri/Cargo.toml -- --check
cargo test --manifest-path src-tauri/Cargo.toml
npm run build:apple-assist-helper:live
npm run build
git diff --check
npm audit
cargo audit --file src-tauri/Cargo.lock
```

The built app must report version `0.12.0`, bundle identifier `lab.hazakura.note`, product name `hazakura editor`, and executable `hazakura-editor`.

The built app must pass:

```bash
codesign --verify --deep --strict --verbose=2 "src-tauri/target/release/bundle/macos/hazakura editor.app"
```

Expected warning for the local ad-hoc app:

```bash
spctl -a -vv -t open "src-tauri/target/release/bundle/macos/hazakura editor.app"
```

Gatekeeper may reject the app with insufficient-context / not-notarized style output because this tag does not produce a notarized distribution asset.
