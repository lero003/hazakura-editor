# v1.3 Daily Trust Follow-up

Status: Implemented; full gates and core built-app smoke passed
Scope: v1.3 daily-use trust and bounded polish
Authority: High
Last reviewed: 2026-07-01

## Decision

v1.3 deepens the existing single-document Safe Editor instead of adding a
new product surface. The source lane contains four independent slices:

1. Save As preserves the same open-tab / CodeMirror session for the same
   language family, including selection, scroll, undo, and redo.
2. Hazakura Local Assist review uses explicit `採用` / `破棄`; accepting
   keeps an unsaved buffer and never saves automatically.
3. Reading Focus TOC shows up to two H3+ subheadings, a remaining count,
   and measured page progress for the current chapter only.
4. PDF export opens an A4 margin chooser with `狭い` (10/10 mm), `標準`
   (18/16 mm), and `広い` (25/22 mm). The preset is request-scoped and
   no arbitrary CSS or print UI is exposed.

The package/app version is now `1.3.0` as the prepared v1.3 candidate. The
signed `1.3.0` pkg has not been built yet; the last built candidate remains
the `1.2.0` build `57` artifact. No v1.3 upload, tag, push, or public
release is implied by this candidate state.

## Boundary Decisions

- Save As keeps the tab id stable while the document path and view-state
  key migrate. A real Markdown-to-HTML/CSS language-family change may still
  remount CodeMirror so the parser stays correct.
- `採用` / `破棄` applies only to the reversible Local Assist transaction.
  File Diff stays read-only; Recovery retains its explicit restore action.
- H3+ TOC text is context, not a new approximate page-jump target.
- PDF stays A4-only. Margin values come from a frontend allowlist while the
  existing Rust main-window, non-empty HTML, parent-directory, and `.pdf`
  destination guards remain unchanged.

The built helper-enabled app also passed these representative checks:

- Save As changed the path without replacing the editing session; Undo after
  Save As reversed the pre-save sentinel and kept the new path dirty.
- Local Assist showed `採用` / `破棄`; `採用` closed review while leaving the
  accepted buffer dirty and unsaved.
- Reading Focus TOC showed H3/H4 context and current measured page text.
- The same manuscript established distinct visible content insets for all three
  595 x 842 point A4 presets. A follow-up found that document `scrollHeight`
  incorrectly added a second blank row of pages. PDF capture now keeps the
  horizontal layout at one A4-high row and derives its used columns from
  rendered text/media origins, so long inline overflow cannot add trailing
  pages. In the built app, the standard-margin export changed from six pages
  (three content plus three blank) to three pages with content on every page.
- Extended RC smoke then reproduced the reported v1.3 regression: native table
  descendants did not paint reliably across the horizontal WebKit columns, and
  fenced code inherited light text after its dark background was removed. The
  export-only HTML now annotates each table's column count, presents each row as
  a fixed-column grid that may continue between A4 columns, and keeps fenced code
  as a dark, width-bounded block. A standard-margin fixture produced five A4
  pages in the built app; macOS Preview showed all six columns in the first table,
  readable fenced and inline code, all rows 01-30 across the long table, and the
  final sentence on the last page. Poppler and `pdfplumber` were also used for
  page shape and text-continuity checks; macOS Preview remains the visual ground
  truth for the embedded Japanese font subsets.

## Extended RC Smoke Still Useful

The scoped behavior is covered by automated tests and the representative app
checks above. Before a v1.3 release candidate, complete the remaining interaction
edges: non-empty selection/scroll/Redo and Save As cancellation; Local Assist
`破棄` plus ordinary File Diff/Recovery labels; the TOC `ほか1件` and chapter-
opener click; and PDF images plus dialog cancellation. These are
release-candidate breadth checks, not open implementation slices.

## Avoid

- Do not turn PDF settings into arbitrary numeric margins, page-size/DTP
  controls, or a macOS print path.
- Do not extend Local Assist acceptance into auto-save or generic Diff apply.
- Do not add inferred H3 page jumps or a second e-book document model.
- Do not bundle release metadata, App Store submission, or packaging into
  source verification without explicit user approval.
