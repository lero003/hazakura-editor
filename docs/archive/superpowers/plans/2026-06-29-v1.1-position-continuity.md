# v1.1 Position Continuity Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.

**Goal:** Reader、Editor、Previewの文書別位置をAppWorkspace所有のregistryへ統合し、タブ／Markdownリンク遷移後もEditor位置を復元したうえで、Recovery確認と署名済みv1.1 pkg作成まで閉じる。

**Architecture:** DocumentViewStateRegistryが文書キーごとのEditor／e-book／Preview位置を保持し、AppWorkspaceが唯一の所有者になる。各surfaceはcontrolledな初期位置とpatch callbackだけを受け取り、Markdown source、保存状態、undo履歴、永続化には触れない。PreviewとRecoveryは再現結果に応じてimplemented、verified no-op、manual-blockedのいずれかで証拠を残す。

**Tech Stack:** React 19、TypeScript 6、CodeMirror 6、Vitest、Tauri 2、Rust、macOS App Store signing scripts

- [ ] **Step 1: patch merge、clamp、pruneの失敗テストを書く**

~~~ts
import { describe, expect, it } from "vitest";
import {
  clampEditorViewState,
  patchDocumentViewState,
  pruneDocumentViewStates,
} from "./documentViewState";

describe("document view state", () => {
  it("patches one surface without dropping the others", () => {
    const current = {
      "/a.md": {
        ebook: { chapterIndex: 2, pageIndex: 1 },
        editor: { anchor: 8, head: 8, scrollRatio: 0.4 },
      },
    };
    expect(
      patchDocumentViewState(current, "/a.md", {
        editor: { scrollRatio: 0.7 },
      }),
    ).toEqual({
      "/a.md": {
        ebook: { chapterIndex: 2, pageIndex: 1 },
        editor: { anchor: 8, head: 8, scrollRatio: 0.7 },
      },
    });
  });

  it("clamps stale selection offsets and scroll ratio", () => {
    expect(
      clampEditorViewState({ anchor: 40, head: -2, scrollRatio: 2 }, 12),
    ).toEqual({ anchor: 12, head: 0, scrollRatio: 1 });
  });

  it("prunes closed documents without changing kept entries", () => {
    const a = { editor: { anchor: 1, head: 1, scrollRatio: 0.2 } };
    expect(
      pruneDocumentViewStates(
        { a, b: { ebook: { chapterIndex: 1, pageIndex: 0 } } },
        ["a"],
      ),
    ).toEqual({ a });
  });
});
~~~

- [ ] **Step 3: 共通型と最小helperを実装する**

~~~ts
export type EditorViewState = {
  anchor: number;
  head: number;
  scrollRatio: number;
};
export type EditorViewStatePatch = Partial<EditorViewState>;
export type EBookViewState = {
  chapterIndex: number;
  pageIndex: number;
  sourceLine?: number;
};
export type PreviewViewState = { scrollRatio: number };
export type DocumentViewState = {
  editor?: EditorViewState;
  ebook?: EBookViewState;
  preview?: PreviewViewState;
};
export type DocumentViewStateRegistry = Record<string, DocumentViewState>;
~~~

patchDocumentViewStateはnested editor patchを既存editorへmergeし、ebook/previewを落とさない。clampEditorViewStateはanchor/headを0..docLength、scrollRatioを0..1へ収める。pruneDocumentViewStatesはkeep keyのSetを使い、残すentry参照を維持する。EBookPaneのEBookReaderLocationはEBookViewStateのtype aliasへ変更する。

const [documentViewStates, setDocumentViewStates] =
  useState<DocumentViewStateRegistry>({});
const activeDocumentViewState = activeEbookDocumentKey
  ? documentViewStates[activeEbookDocumentKey] ?? null
  : null;

const handleEditorViewStateChange = (patch: EditorViewStatePatch) => {
  if (!activeEbookDocumentKey) return;
  setDocumentViewStates((current) =>
    patchDocumentViewState(current, activeEbookDocumentKey, { editor: patch }),
  );
};
~~~
