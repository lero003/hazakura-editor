# v1.3 Daily Trust Design

Status: Review
Scope: v1.3 daily-use trust and bounded polish
Last reviewed: 2026-07-01

## Goal

v1.3 は、機能面を広げるのではなく、毎日の執筆で迷いや位置喪失を減らす。
次の4件を相互に依存しない小さなスライスとして実装し、それぞれを個別に
テスト・レビュー・取り消せる状態にする。

1. Save As 後も同一タブの Editor session を維持する。
2. Hazakura Local Assist のレビュー結果を明示的な `採用` / `破棄` にする。
3. e-book Mode の目次に章内の文脈と現在位置を追加する。
4. A4 PDF export に `狭い` / `標準` / `広い` の余白プリセットを追加する。

## Product Boundary

- Markdown/text source を正本のままにする。
- Save、AI proposal、e-book Mode、PDF export は引き続きユーザーの明示操作で
  開始する。
- `採用` は buffer の変更を保持するだけで、自動保存しない。
- 通常の file Diff、disk comparison、Recovery review に、実在しない
  `採用` / `破棄` の意味を追加しない。
- e-book 目次は表示・移動レイヤーに限定し、source、見出し、章構造を変更しない。
- PDF は A4 固定のままとし、自由な数値入力、用紙サイズ、印刷 UI、DTP layout
  機能へ広げない。
- Git、LSP、terminal、任意コマンド、plugin、project-wide indexing、外部AI/API、
  auto-apply、auto-save、Preview DOM editing を追加しない。
- Agent Workbench と App Store lane の境界を変更しない。

## Considered Release Shapes

### 1. Core trust only

Save As continuity と Local Assist review clarity だけを実装する案。技術的な集中度は
高いが、Reader / Export で既に記録済みの小さな不満が残り、v1.3 の体感変化が弱い。

### 2. Visible polish only

Local Assist、e-book 目次、PDF余白だけを実装する案。UI上の変化は分かりやすいが、
Save As で undo / position を失う既知の安心感の穴を残す。

### 3. Four independent trust slices

採用案。4件を同じ大きなリファクタへまとめず、Core、Review、Reader、Export の順に
独立実装する。同一releaseに入っても、各sliceのacceptanceとregression testは分ける。

## Slice 1: Save As Session Continuity

### Behavior

pathless tabまたは別名保存対象のtabがSave Asに成功し、同じtab idのままpathだけを
得た場合、Editorのselection、scroll、undo、redoを維持する。Save As失敗、cancel、
既存open tabとのpath衝突では、現在のbufferとsessionを変えない。

別文書へのtab切替は従来どおり別sessionとして扱う。`.md`から`.html`のようにSave Asで
language familyが変わる場合は、正しいparserへ切り替えるためremountを許す。この例外では
selection / scrollを可能な範囲で復元するが、CodeMirror undo履歴の維持は保証しない。

### Architecture

`documentKey` の二つの責務を分離する。

- `documentKey = activeTab.path || activeTab.id`: path、language、document view-state
  registryのidentity。
- `editorSessionKey = activeTab.id`: 同じopen tab内のCodeMirror session identity。

`EditorPane` のremount判定は `editorSessionKey` とlanguage familyを使う。Save Asで
pathだけが変わる通常MarkdownケースではCodeMirrorを破棄しない。view-state registryは
Save As成功時に旧document keyの状態を新keyへ移し、Editor / Preview / e-bookの位置が
孤立しないようにする。rekeyは成功後だけ行い、失敗やcancelでは何もしない。

## Slice 2: Explicit Local Assist Review Result

### Behavior

Hazakura Local AssistがbufferへAI edit transactionを適用した後のreview barに、
次の明示actionを置く。

- `採用`: 現在のbufferを保持し、pending review transactionだけを閉じる。
- `破棄`: transactionの`beforeBuffer`へ戻し、pending review transactionを閉じる。
- `差分を開く` / `差分を閉じる`: review中の表示切替として維持する。

`採用` 後もtabはdirtyのままにし、Saveは別の明示操作とする。ボタン近傍またはtooltipで
`採用しても未保存` と分かるcopyを出す。曖昧な `×` はreview decisionには使わない。

### Scope Guard

この変更は`AppleAssistReviewBar`のAI edit transactionだけに適用する。
`FileCompareView`は読み取り専用の`閉じる`、Recovery backupは
`このバックアップを復元`、disk/draft/conflict comparisonは既存の意味を維持する。
Local Assistの`採用`はmodel出力の正しさを保証する表現ではなく、ユーザーが現在の
unsaved bufferを残す判断を表す。

## Slice 3: Richer e-book Table Of Contents

### Behavior

Reading Focusの目次で、H1 / H2 chapter rowに次の情報を追加する。

- chapter titleと現在のchapter index。
- 章内にH3以下がある場合、その先頭2件の小見出し名。
- 3件以上ある場合は残数を`ほかN件`として示す。
- 現在章だけ、現在page / measured page countを示す。

chapter rowのclickは従来どおりchapter openerへ移動する。H3以下は文脈表示に留め、
誤差のあるpage推定jumpを新設しない。heading-like text inside fenced code blocks、YAML
frontmatter、Setext headingは既存のchapter parser規則に従い、目次だけ別解釈しない。

### Architecture

既存の`splitMarkdownIntoChapters(source)`が返すraw heading segmentsから、
`coalesceChaptersToTopLevel`後の各chapterに属するH3以下のlabelを導くpure helperを置く。
rendered HTMLやDOMから見出しを再抽出しない。sourceを二度解釈する新parserも作らない。

未計測chapterのpage countは表示しない。現在章の`measuredPageCount`だけを使い、
非表示章に推測値を出さない。章移動、page navigation、reader location persistence、
spread behaviorは変更しない。

## Slice 4: A4 PDF Margin Presets

### Behavior

`PDFとして書き出す…` を選ぶと、保存先選択の前にA4余白presetを選ぶ小さなdialogを開く。

| Preset | Top / Bottom | Left / Right |
|---|---:|---:|
| 狭い | 10 mm | 10 mm |
| 標準 | 18 mm | 16 mm |
| 広い | 25 mm | 22 mm |

`標準`をdefaultとし、現在の`@page { margin: 18mm 16mm; }`と同じ見た目を維持する。
選択は当該export requestだけに適用し、Preferencesや永続設定を追加しない。dialog cancel、
save dialog cancel、export中のdocument変更は既存どおり何も書き出さず終了する。

### Architecture

frontendに`PdfMarginPreset`とpreset-to-CSS-valueのpure mappingを置き、選択値から
standalone export HTMLの`@page` marginを生成する。HTMLへ埋め込む値はallowlistされた
presetからだけ作り、任意CSS文字列をRust commandへ渡すUIを作らない。

Rust側のmain-window label、non-empty HTML、existing parent、`.pdf` extension検証と、
WebKit / PDFKit内のA4 page rect生成は維持する。focused testとbuilt PDF inspectionで
WebKitが各pageの`@page` marginを反映することを確認する。反映されない場合は、A4 page
rectやsecurity boundaryを推測で変更せず、同じpreset契約を保つexport-only HTML / CSSの
実装方法へ切り替え、actual PDFで再確認する。

## Data And Error Flow

- Save As: choose path -> validate collision -> write -> update tab path -> rekey view state。
  write完了前にsession identityやregistryを変更しない。
- Local Assist: explicit generate -> AI edit transaction -> Diff review -> `採用`または`破棄`。
  どちらもSaveを呼ばない。
- e-book TOC: canonical source -> existing chapter split -> top-level chapters + subheading metadata
  -> display only。sourceへ逆方向のwriteはない。
- PDF: choose preset -> build allowlisted export CSS -> choose `.pdf` path -> validate document
  identity -> invoke existing guarded Rust export command。

既存エラーは既存status / global error経路を使う。新しい失敗を黙って成功扱いしない。
Save As失敗時はbufferをdirty/recoverableのままにし、PDF失敗時は不完全な成功messageを
出さない。

## Testing

各sliceはTDDでfocused regressionを先に追加する。

### Save As

- Markdown pathless tabのSave As後にEditorView instanceをremountせず、undo / redoが効く。
- selectionとscrollが維持される。
- path更新後、旧view-state keyが孤立せず新keyへ移る。
- cancel、write failure、open-tab collisionではsessionとregistryを変えない。
- language family変更時だけremountし、selectionをclampして復元する。

### Local Assist

- `採用`はbufferを変えずreview transactionだけをclearする。
- `破棄`は`beforeBuffer`へ戻しreview transactionをclearする。
- `採用`後もdirty状態と明示Save boundaryが残る。
- file Diff / Recoveryのaction labelsとbehaviorは変わらない。
- Japanese、English、kana copyとkeyboard focusが成立する。

### e-book TOC

- H1 / H2 chapterへH3以下の先頭2件と残数が対応する。
- fenced code / frontmatter / Setextを誤って小見出しにしない。
- 現在章だけ実測page progressを表示する。
- chapter click、page navigation、location persistenceは既存挙動を維持する。

### PDF

- 3 presetが正しいallowlisted CSS marginへ変換される。
- defaultが現行相当の`18mm 16mm`である。
- dialog cancelとsave cancelはRust exportを呼ばない。
- document変更guard、`.pdf` validation、main-window-only commandを維持する。
- actual multi-page PDFで各pageの余白差を目視またはpage render比較し、ファイル生成だけで
  余白機能を通過扱いにしない。

focused tests後に、現行docsに従って少なくとも次を実行する。

```bash
npm run test
npm run build:vite
cargo fmt --manifest-path src-tauri/Cargo.toml -- --check
cargo test --manifest-path src-tauri/Cargo.toml
npm run build
git diff --check
```

UI behaviorはbuilt appでSave As、Local Assist review、e-book TOC、PDF preset dialogを
個別に確認する。PDFは生成物をrenderして余白を確認する。実際に操作していない項目を
manual smoke passedとは報告しない。

## Implementation Order

1. Save As session continuity。
2. Local Assist explicit review result。
3. e-book TOC context。
4. PDF margin presets。
5. Full verification、built-app / PDF smoke、docs sync。

前sliceの失敗を次sliceの変更で隠さない。各sliceのfocused testsがgreenになってから次へ
進む。依存追加、version bump、App Store build-number変更、pkg生成、upload、tag、push、
publicationはこの設計の対象外とする。

## Completion Criteria

- Save As後の通常Markdown tabでselection、scroll、undo / redoが維持される。
- Local Assist reviewが明示的な`採用` / `破棄`となり、採用しても自動保存されない。
- e-book目次で章内小見出しと現在章の実測page progressが読める。
- A4 PDFで3余白presetが選べ、actual multi-page PDFに差が反映される。
- 4件が独立したfocused regressionで固定され、full source/build gatesが通る。
- `docs/current-work.md`、`docs/current-status.md`、`docs/roadmap.md`、
  `docs/v1.1-v1.2-followup.md`または後継v1.3 doc、`docs/smoke-checklist.md`、
  `docs/handoff.md`が実装済みstateと残リスクに同期する。
- 既存の`src-tauri/tauri.conf.appstore.json` build-number変更を含む、依頼外のworktree変更を
  巻き込まない。
