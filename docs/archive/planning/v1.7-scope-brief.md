# v1.7 Scope Brief / 依頼ブリーフ

Status: Active scope brief for separate v1.7 requests
Scope: The complete v1.7 product and quality boundary, not an implementation slice
Authority: High for what v1.7 must deliver; execution order remains in `docs/archive/operations/v1.7-trust-scale-plan.md`
Last reviewed: 2026-07-11

## 1. 依頼の目的

v1.6で入った Import Assist を、日常的に使える **Reference Compare** 体験へ
仕上げる。v1.7は新しい大きな製品面を増やす版ではなく、
「原本を見ながら Markdown を直し、モードを往復しても、長文でも、未保存でも
失わない」状態を完成させる版とする。

v1.7のユーザー向け約束は次の一文に集約する。

> 参照資料を右のプレビューとして開き、中央の Markdown を見比べながら直せる。

この約束を、編集の継続性、草稿の復旧、長文時の失敗の明瞭さ、アクセシビリティ、
配布版での再現性まで含めて成立させる。

## 2. v1.7完了後の体験

利用者は、次の一連の流れを一つの Safe Editor 体験として完了できる。

1. Markdown を中央の主編集面で開く、または PDF / 画像 / Markdown / text を
   右側の読み取り専用参照として開く。
2. Import Assist で作った未保存 Markdown 草稿について、元資料を右に残したまま
   内容を編集する。ページ追従や `要確認` は補助情報として明示的に扱う。
3. Preview、L Mode、e-book、Outline、Diff、Reference の間を移動しても、
   Undo、カーソル、IME、選択、長文スクロール、比較文脈を不用意に失わない。
4. 新規または Import Assist のパスレス草稿が異常終了後に見つかった場合、
   元ファイルを勝手に上書きせず、利用者が確認して復旧できる。
5. 大きな差分、検索、PDF / 参照画像、書き出しで上限に達した場合、固まるのではなく、
   キャンセル・省略・再試行の意味が分かる状態になる。
6. キーボード、VoiceOver、拡大表示、コントラスト、Reduce Motion でも、
   主要な編集・参照・モード切替を完了できる。
7. App Store 配布形状のアプリで、上記の境界が単体テストだけでなく実機相当の
   Reference Compare 操作として確認できる。

## 3. スコープに含めるもの

### A. Reference Compareの製品化

- 編集可能な Markdown は常に中央 / 主列に一つだけ置く。
- PDF、画像、Markdown、text の参照を右側のプレビュー風ペインに一つだけ置く。
- Import Assist の自動ペアリング、ページ追従、一時停止 / 再開、ページ単位の
  `要確認` を、明示的かつ助言的な機能として整える。
- ワークスペース、外部ファイル、File メニュー、Command Palette、既存タブからの
  明示的な参照導線を一貫させる。
- 参照の置換・終了・削除・読み込み失敗が、編集バッファや保存状態を変更しないようにする。

### B. 編集継続性とデータ信頼性

- L Mode を表示レイヤーとして安定させ、通常表示へ戻った際にサイドペイン、
  比較文脈、Undo、カーソル、IME、スクロール位置を不用意に失わない。
- パスのない新規 / Import Assist 草稿を、ソースファイルとは分離したアプリ内の
  期限・容量制限付き復旧候補として扱う。
- Dirty close、Save As、外部変更、参照ファイル置換 / 削除の状態を、利用者が判断できる
  明示的な確認として保つ。
- 「ソースへの自動保存なし」と「クラッシュ後の復旧なし」を同じ意味にしない。

### C. 長文・大容量時の境界と失敗耐性

- 草稿保存、Diff、ワークスペース検索、PDF / EPUB 書き出し、参照画像 raster の
  メモリ・サイズ・時間・DOM負荷を個別に制限する。
- キャンセル可能な処理はキャンセルでき、制限到達時は切り捨て理由や再試行方法を示す。
- 長い text 参照は、日本語折返し・全文選択・スクロール位置を壊さないことを優先する。
  可変行高を正しく計測・補正できるまで固定高 windowing / virtualize は使わず、全文を
  DOM に保つ。editor共通の10 MB file-open上限とは別に、text referenceは150万文字または
  5万論理行を上限とし、超過時は既存参照を置換せず理由を表示する。5,000行の折返しと
  budget超過の実機確認が終わるまで、大容量境界を完了扱いにしない。
- フロントエンド例外時に編集セッションを救済する root-level recovery surface を用意する。
- Rust / frontend / integration のテスト結果が、単独再実行だけで緑に見える状態を残さない。

### D. 発見性とアクセシビリティ

- Start Panel で「書く・読む・確かめる」の価値を短く説明する。
- Preview、L Mode、e-book、Outline、Diff、Reference を機能名だけでなく用途で理解できる
  小さな初回ヒントを用意する。常時表示のChromeは増やさない。
- Workspace rename、ペイン操作、ページ操作、モーダル、モード切替をキーボードと
  VoiceOverで扱えるようにする。
- コントラスト、拡大文字、Reduce Motion、IME、長い日本語折返しを主要面で確認する。
- Assist が利用不可 / オフのときも、オンデバイス・明示適用・保存前確認の境界を短く説明する。

### E. リリース準備とドキュメント整合

- Reference Compare の App Store / sandbox / picker / CSP / narrow window / keyboard を含む
  パッケージ検証を完了する。
- README、current-status、current-work、roadmap、handoff、リリース文言が、実装済みと
  未実装を過大表現せず、v1.7の完成条件を同じ意味で示す。
- v1.6 (`1.6.0`) は、v1.7の完成条件と明示承認が揃うまで公開済み版として維持する。

## 4. 明示的に含めないもの

Book Scope Alpha は v1.7 のスコープに含めない。単一ドキュメントと Reference Compare の
ループを実証した後の v2 判断とする。

その他、v1.7では次を追加しない。

- 二つの編集可能ペイン、汎用 split editor、Review Desk の復活
- 汎用 AI チャット、provider plugin、cloud OCR、background rewriting
- Git、LSP、terminal、任意コマンド実行、project-wide indexing
- 自動適用、自動保存、自動コミット、アプリ内のリリース自動化
- 縦書きを新しい構造的プロダクト面として先行実装すること

Agent Workbench は引き続き Developer / GitHub-only の別境界であり、Safe Editor の
v1.7体験へ取り込まない。

## 5. 完了条件（スコープベース）

v1.7を完成扱いにするには、個別スライスの完了数ではなく、次の状態をまとめて確認する。

- 中央の Markdown だけが編集・保存対象で、右の参照は常に読み取り専用である。
- Reference Compare が PDF / 画像 / Markdown / text と Import Assist の実運用経路で成立する。
- モード切替、タブ、Save As、Dirty close、外部変更、参照置換で文書の同一性と利用者の意図が保たれる。
- 未保存パスレス草稿に明示的な復旧経路があり、ソースファイルの無断変更がない。
- 大容量処理が上限内で完了するか、失敗しても理由と次の操作が分かる。
- キーボード / VoiceOver / コントラスト / 拡大 / Reduce Motion の主要確認が通る。
- パッケージ版での Reference Compare 証跡、テスト結果、既知の残リスクが揃う。
- READMEとリリース文言が実装状態を誇張せず、v1.7の境界とBook Scope Alphaの延期を説明する。
