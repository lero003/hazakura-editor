## 概要

v1.8.0 (build 89) がApple審査通過してリリースされたため、ドキュメントを「公開版 = 1.8.0」へ一括更新し、GitHubタグ `v1.8.0` とGitHub Release（source-only）を作成します。

## 作業フロー

### Step 1: リリース前プレチェック
`docs/release-pre-check.md` に従い、以下を確認（ストップ条件が出なければ次へ）:
- 作業ツリーがクリーンであることを確認済み
- `git diff` 範囲でローカルパス・不適切内容・シークレットのgrep
- バージョンメタデータ (`package.json` / `tauri.conf.json` / `Cargo.toml`) がすべて `1.8.0` であることを確認済み

### Step 2: ドキュメント一括更新（7ファイル）
全ファイルで「1.8.0はTestFlight候補・公開版は1.7.0」→「公開版は1.8.0」に更新します。

### Step 3: commit・タグ作成・push
- 変更ファイルのみstage（`docs/` と `README.md`）

- annotated tag `v1.8.0` を作成（過去のv1.2〜v1.4タグの慣行に合わせ、タイトル行＋日本語リリースノート本文）

### Step 4: GitHub Release作成（source-only）
- `gh release create v1.8.0` でReleaseページを作成
- リリースノートは `1.8.0-app-store-release-notes.md` のApp Store Connect Copyをベースにした本文
- バイナリアセット（DMG/pkg）は添付しない source-only release
- `--latest` フラグでlatest Releaseに設定（現在のlatestが古いDMG previewの場合のため）
