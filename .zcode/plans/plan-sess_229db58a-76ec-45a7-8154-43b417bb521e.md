## 方針

v1.7 Reference Compare + 信頼性強化スライス（T-1 L Mode 連続性 / T-2 pathless 草稿復旧 / S-1 processing budgets / S-2 失敗明示 / S-3 wrap-safe 長文参照 + budget / S-4 発見性）のソースが揃い、ローカルゲートは全て緑（Vitest 157 files / 1361 tests、Rust 340/340、typecheck、build:vite、fmt、diff --check）。Explore エージェントによる詳細レビューでも「ブロックするバグ・未完成ロジック・データ安全性の後退は見当たらない」と確認済み。

TestFlight を「v1.7 完成前の実機テスト用配信」と位置づけ、バージョンを 1.6.0 → 1.7.0 に揃えて push する。App Store build counter（`tauri.conf.appstore.json` の `bundleVersion: "83"`）は今回は触らず、実際の配信時にユーザーが調整する。

docs では引き続き「v1.7 は TestFlight 実機テスト段階であり、App Store 公開版は 1.6.0 のまま」を誇張なく明記する。

### 6. コミット（2分割）
**コミットA（ソース変更）**: すでに作業済みの v1.7 ソース変更（src/ 配下の M / ?? ファイルすべて）+ docs の変更（手元で更新がかかっている分）。

### 8. TestFlight ビルド手順の提示（実行はユーザー）
push 後、ユーザーが TestFlight 用 pkg を作るためのコマンドを案内：
