# AGENTS.md

## Role

このプロジェクトでは、変更を小さく保ち、Markdown-first safe editor という軸を守る。

## Working Rules

- まず目的を確認し、現状の文書とコードを読む
- 実装範囲をMVPに寄せる
- 任意コード実行、Git操作、LSP、ターミナル、拡張機能の追加へ広げない
- セキュリティ境界に関わる変更は `docs/security-boundary.md` を先に確認する
- ドキュメント判断は `README.md` と `docs/` 配下を優先し、`docs/markdown-safe-editor-plan.md` は発想の原本として扱う
- テストまたは確認コマンドを実行してから完了報告する

## Reporting

作業後は、変更したこと、確認したこと、残っているリスク、次に見るべきことを簡潔に報告する。

# Agent Instructions

## Language

- このプロジェクトでは、ユーザー向けの説明・進捗報告・質問・レビューコメントは日本語で行う。
- コード、API名、型名、ファイル名、コマンド、エラーメッセージは原文の英語を維持してよい。
- 英語のログやエラーを引用する場合は、必要に応じて日本語で要約する。
- コミットメッセージやPRタイトルは、明示がない限り日本語で簡潔に書く。

## Communication Style

- まず結論を短く述べる。
- 変更前に大きな方針を一言で説明する。
- 実装後は「変更点」「確認したこと」「残タスク」を簡潔にまとめる。
- 長い英語の説明を避け、日本語で短く報告する。
- 不確実な点は推測で進めず、「未確認」「要確認」と明示する。

## Work Policy

- 既存の設計・命名・UIトーンを尊重する。
- 不必要な大規模リファクタリングを避ける。
- 変更はできるだけ小さく、レビューしやすい単位に分ける。
- 既存の挙動を変える場合は理由を説明する。
- 破壊的操作、依存追加、設定変更、ファイル大量削除の前には確認する。

## Project Notes

- このプロジェクトは日本語ユーザーを主対象にする。
- UI文言やヘルプ文は自然な日本語を優先する。
- 専門用語は必要に応じて英語併記してよい。
